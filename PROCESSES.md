# 리안 컴퍼니 Core Processes

> **EOS 기반. 이 문서에 있는 프로세스는 리안의 명시적 승인 없이 변경 불가.**
> AI가 "개선"이라는 명목으로 프로세스를 바꾸는 것을 방지하기 위한 문서.
> 변경하려면: 변경 제안서 작성 → 리안 컨펌 → 실행 → 검증

---

## 1. 신규 프로젝트 프로세스

```
리안 아이디어 → 시은 명확화 (최대 2번 질문)
  → main.py (이사팀: 태호+서윤 병렬 → 민수 전략 → 하은 검증 → 토론 → 준혁 GO/NO-GO)
  → GO면: build_team.py (교육팀이 팀 자동 생성)
  → 팀 실행 (run_{팀명}.py)
  → 보고사항들.md에 결과 저장
```

**변경 불가 규칙**:
- 이사팀 순서 변경 금지 (태호+서윤 → 민수 → 하은 → 토론 → 준혁)
- 준혁 GO/NO-GO 없이 팀 생성 금지
- 팀은 반드시 build_team.py로 생성 (수동 생성 금지)

---

## 2. 영업 프로세스 (오프라인)

```
타겟 발굴 → 네이버 플레이스 진단 → 진단 보고서(PDF/PPT) 생성
  → 1차 DM (카톡/문자) → 답장 대기
  → 답장 시: 2차~6차 메시지 시퀀스 (거절 유형별 대응)
  → 관심 표현 시: 제안서 + 가격표 발송
  → 클로징 → 온보딩
```

**변경 불가 규칙**:
- 리안은 대면 안 나감. 비대면(카톡/문자)으로만 진행
- 진단 보고서는 반드시 "위기감 → 신뢰 → 의존성" 3단계 구조
- 패키지: 주목(29만), 집중(49만), 시선(89만) — 리안 승인 없이 가격 변경 금지

---

## 3. 콘텐츠 납품 프로세스

```
클라이언트 온보딩 → 업종/키워드 분석
  → 콘텐츠 전략 수립 (서진호: SEO 키워드)
  → 콘텐츠 제작 (한서연: 블로그, 박지우: 인스타, 최도현: 광고카피)
  → 김태리 QC → 클라이언트 확인 → 발행
  → 정민재: 월간 성과 리포트
```

**변경 불가 규칙**:
- 납기 준수 100% (기한 넘기면 클라이언트 이탈)
- 리포트는 반드시 숫자 중심 ("이래서 돈이 됐다")
- 클라이언트 확인 단계 스킵 금지

---

## 4. 자체 마케팅 프로세스

```
매일: 인스타 캡션 1개 + 블로그 1개 (ops_loop.py daily)
매주: 성과 리뷰 + 방향 수정 (ops_loop.py weekly)
메타광고: 소액(3~5만원) 테스트 → 성과 확인 → 확대/중단 결정
```

**변경 불가 규칙**:
- 콘텐츠는 AI 작성 + 리안이 수동 업로드 (자동 포스팅은 약관 위반)
- 메타광고 예산 증액은 리안 승인 필요
- 콘텐츠 컨셉은 각 팀의 mission.md에 따름

---

## 5. 시스템 변경 프로세스

```
문제 감지 (AI 또는 리안)
  → 변경 제안서 작성 (뭘, 왜, 어떻게)
  → 리안 컨펌
  → 실행
  → 검증 (돌려보고 문제없는지 확인)
```

**변경 불가 규칙**:
- **AI가 독단적으로 시스템 구조를 바꾸지 못함**
- 에이전트 프롬프트 수정 → 리안 컨펌 필요
- CLAUDE.md, OPERATIONS.md, PROCESSES.md, company_context.md 수정 → 리안 컨펌 필요
- 코드(pipeline.py, agents/*.py) 구조 변경 → 리안 컨펌 필요
- mission.md 내용은 리안 또는 해당 팀이 분기 리뷰 시 수정

## 주간 Scorecard 지표

매주 Level 10에서 이 숫자들을 체크한다.

| # | 지표 | 담당팀 | 목표 | 측정 방법 |
|---|------|--------|------|-----------|
| 1 | 영업 DM 발송 수 | 오프라인마케팅 | 20건/주 | 리안 수동 입력 |
| 2 | DM 답장률 | 오프라인마케팅 | 30% | 답장 수 / 발송 수 |
| 3 | 제안서 발송 | 온라인영업 | 3건/주 | 자동 카운트 |
| 4 | 블로그 발행 수 | 온라인납품/마케팅 | 3건/주 | 발행 카운트 |
| 5 | 인스타 게시물 | 온라인마케팅 | 3건/주 | 게시물 카운트 |
| 6 | 신규 계약 | 전체 | 1건/월 | 리안 수동 입력 |
| 7 | 클라이언트 유지 | 온라인납품 | 100% | 이탈 여부 |

🟢 목표 달성 | 🟡 80% 이상 | 🔴 80% 미만
🔴 3주 연속 → IDS 강제 트리거 (피벗 검토)

---

## 주간 Level 10 Meeting 아젠다

리안이 매주 1회(15~30분) 확인하는 구조.
AI가 1~5번 자동 생성 → 리안은 6번(IDS)에만 집중.

```
1. Good News (2분) — 이번 주 잘된 것
2. Scorecard 리뷰 (3분) — 🔴 항목 중심
3. Rock 진척도 (3분) — On Track / Off Track
4. Headlines (2분) — 특이사항, 시장 변화
5. To-Do 리뷰 (2분) — 지난주 약속 완료 여부
6. IDS (15분) — 🔴 항목 처리:
   - Identify: 뭐가 문제?
   - Discuss: 원인은?
   - Solve: 다음 주 액션은?
7. 마무리 (1분) — 다음 주 To-Do 확정
```

## 분기 사이클 (90일 Rock)

```
Week 0: 분기 계획
  - 지난 분기 Rock 달성률 리뷰
  - 새 Rock 3~5개 설정
  - 각 팀에 Rock 배분
  - Scorecard 지표 조정 필요하면 조정

Week 1~12: 실행
  - 매주 Level 10
  - 🔴 3주 연속 → ops_loop.py pivot 실행

Week 13: 분기 리뷰
  - Rock 완료율 체크 (80% 이상 = 성공)
  - V/TO 업데이트 필요하면 리안과 논의
  - 다음 분기 Rock 준비
```
