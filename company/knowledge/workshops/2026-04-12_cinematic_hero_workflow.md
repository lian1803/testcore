# 시네마틱 히어로 워크플로우 — 핵심 원칙

출처: 리안 (2026-04-12)

---

## 목표 (딱 두 가지)

1. **브랜딩이 사용자로 하여금 이해가 되느냐**
2. **그걸 충분히 사용자의 시선을 사로잡을만큼 퀄리티가 좋으냐**

---

## 에이전트 체인

### 1) Director — 은유 선택
- Input: 주제 (예: "디스커버리")
- 3~7개 은유 나열 (탐험, 동굴, 여행, 지도, 신호, 발굴, 등대 등)
- **1개 선택** — 브랜드 스토리에 가장 맞는 것
- 영화 오프닝처럼 짧은 시네마틱 스토리 (5~10문장)
- Output: HERO_CONCEPT.md
- **코드 금지. 기술 선택 금지. 은유와 스토리가 먼저.**

### 2) Visual Director — 스토리보드화
- Input: HERO_CONCEPT.md
- 스토리 → 비주얼 플랜:
  - 첫 프레임 (사용자가 처음 보는 것)
  - 3~5초 모션 시퀀스
  - 텍스트 등장 타이밍과 방식
  - 브랜드/로고 드러나는 시점
  - 사용할 이펙트 (crack, wave, glow, unfold, particles, video 등)
  - 기술 선택 (ai_video / glb_model / css_motion / three_js_code)
- Output: HERO_PLAN.md

### 3) Hero Builder — 히어로만 구현
- Input: HERO_PLAN.md
- **히어로 섹션만** 빌드. 은유/스토리/카피 변경 금지.
- 히어로가 반드시:
  - 영화 트레일러 오프닝처럼 느껴져야 함 (잔잔한 배경 금지)
  - 브랜드의 아이디어와 감정을 명확히 전달
  - 1~2초 안에 시선 사로잡기
  - 전체화면 강렬한 모션 이벤트 최소 1개
  - 텍스트와 비주얼 동기화 (random floating text 금지)
- 금지: flat plane + basic material / 제네릭 도형 + 약한 애니메이션

### 4) Page Builder — 나머지 섹션
- 히어로 완성 후, 2/3/4... 섹션 빌드
- 3D 있어도 되고 없어도 됨 — 깔끔하게
- 이유: 보여주고 싶은 브랜딩은 히어로에서 이미 다 했으니까
- 역할: 오퍼 설명, 실적/숫자/증거, 브랜드 서포트
- **히어로와 주목도 경쟁 금지**

---

## 가드레일

- 단일 에이전트가 새 히어로 컨셉을 혼자 발명 금지
- 단일 에이전트가 히어로 + 바디 섹션 전부 한번에 빌드 금지
- Hero Builder가 아니면 히어로 건드리지 마라
