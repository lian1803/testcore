### 1단계 (Pain Point)

| 항목 | 우리 자동 (서윤 2-pass) | 수동 (웹 Claude + Perplexity) |

Validator는 100% PASS인데 실제 깊이는 최악. **구조 검증만으로는 깊이 측정 불가** 확인.

### 1차+2차 공통 Gap (구조적 — 둘 다 동일)

1. Pain Point 양: 자동 16~30 vs 수동 50
2. 5-Layer 깊이: 자동 bullet vs 수동 ASCII+table
3. OKR 섹션 부재
4. 필터링 과정 미출력
5. Problem/STP/VPC 부재
6. Pain Point 역추적 #번호 없음

**→ 모든 gap이 두 케이스에서 동일하게 발견. 도메인별 차이 아님 = 구조적 문제.**

## 🏗️ 아키텍처 옵션 (결정 대기)

### Option A: 민수 프롬프트 키우기 (in-place)
- 기존 민수에 OKR/필터/클러스터/STP/VPC 추가
- Pros: 간단. 새 에이전트 없음
- Cons: GPT-4o max_tokens 8k 한계. 프롬프트 터지면 깊이 오히려 떨어짐
- 예상 도달 품질: **80~85%**

### Option B: 2-layer 아키텍처 (기획 + 설계기획팀 분리) ← 리안 제안
```
[기획팀] 서윤 → 민수(slim) → 하은 → 준혁
        ↓ 1순위 pick + pain pool
[설계기획팀] OKR → 필터/클러스터 → BM(Problem/STP/VPC) → 지훈 PRD
```
- 설계팀이 full pain pool + 1순위 context로 "긴 사고 흐름" 복제
- Pros: 수동 웹 Claude가 이기는 진짜 이유(긴 context)를 복제 가능
- Cons: 에이전트 4개 추가, orchestration 변경, 테스트·디버깅 부담
- 예상 도달 품질: **85~92%**

### 95% 못 가는 이유 (공통)
**도메인별 human intuition**: 수동은 리안이 중간에 방향 틀 수 있음.
자동은 이 체크포인트 없음 → 95% = 근본적 한계.
95% 가려면 "중간 체크포인트"가 필요 (사람 개입).

## 🧭 종합 판단 — 최종 결정

### 결정: **Option B (설계기획팀 분리)**

**근거**:
- 1차+2차 모두에서 **동일한 6개 gap 발견** → 구조적 문제
- 도메인별 차이 아님 → 프롬프트 조정(Option A)으로는 못 잡힘
- Option A의 한계: 민수 prompt에 OKR+필터+클러스터+STP+VPC 다 추가해도 GPT-4o max_tokens 8k 초과 → output 쪼그라듦 (이미 관찰됨)

### 최종 아키텍처

```
[기획팀 — 탐색, 기존]
서윤(pain 50개 목표) → 민수(slim: 스코어링+1순위) → 하은 → 준혁

              ↓ (full pain pool + 1순위 + 반론 전달)

[설계기획팀 — 심화, 신규]
├── agent1: OKR 설계 (O + KR1/2/3)
├── agent2: 필터·클러스터링 (3단계 filter → TOP 5 → #1 deep)
├── agent3: BM 설계 (Problem Statement + STP + VPC + Positioning)
└── 지훈(이관): PRD + Aha Moment + JTBD (context 확장)
```

**핵심 트릭**: 설계팀 3명 각자 Claude Sonnet + **full pain pool (50개) context**.
각 max_tokens 8k → 총 24k output 가능. 민수 혼자 8k에 구겨넣던 것 대비 3배 여유.

### 기획팀 쪽 병행 개선

1. 서윤 Pass 1 pain point 목표 30 → **50개**
2. 서윤 Pass 1 프롬프트에 "압축 금지" 재강화 (1단 인라인 형식으로 16개 나온 버그 재발 방지)
3. 서윤 Pass 2 5-Layer에 ASCII IA diagram + 표 형식 요구
4. 민수는 스코어링+1순위 pick만 담당 (OKR/필터/BM/STP 다 설계팀으로 이관)

### 예상 도달 품질
- 수동 100% 대비 **85~92%**
- 95% 못 가는 이유: 수동은 사람 중간 개입 가능, 자동은 불가

### 95% 넘으려면 (장기)
- 중간 체크포인트 추가: 기획팀 끝나고 리안이 방향 확인 → 설계팀 진행
- 이미 `.pending_questions.json` 구조 있음 → 활용 가능
