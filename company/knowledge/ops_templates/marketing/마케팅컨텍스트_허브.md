# 마케팅 컨텍스트 허브

이 문서는 **서비스별 마케팅 컨텍스트를 한 곳에 정리**하는 허브 템플릿이다.
카피라이터, SEO, SNS, 영업 에이전트 모두 이 파일을 먼저 읽고 작업한다.

## 사용법

1. 새 서비스 런칭 시 이 템플릿으로 컨텍스트 파일 생성
2. `marketing_config.json`의 프로젝트와 1:1 매칭
3. 파일 위치: `company/knowledge/agents/marketing_context_{서비스명}.md`
4. 마케팅 에이전트 실행 시 `context_loader.py`가 자동 주입

## 12 섹션 컨텍스트 템플릿

아래를 채워서 서비스별 컨텍스트 파일을 만든다:

---

### 1. 제품 개요
```
서비스명: 
한 줄 설명: 
카테고리: 
URL: 
현재 상태: [개발중 / beta / 런칭 / 운영중]
```

### 2. 타겟 고객 (ICP)
```
주요 타겟: 
  - 직업/역할: 
  - 회사 규모: 
  - 업종: 
  - 경험 수준: 
  
2차 타겟: 
  - ...
```

### 3. 페인포인트 (고통)
```
1순위: [가장 큰 고통 — 이걸로 훅을 만든다]
2순위: [두 번째 고통]
3순위: [세 번째 고통]

고객이 쓰는 말: "..." (실제 표현)
```

### 4. 해결 방법
```
핵심 혜택 1: [기능 X → 혜택으로 변환]
핵심 혜택 2: 
핵심 혜택 3: 

Before: [사용 전 상태]
After: [사용 후 상태]
```

### 5. 경쟁/대체재
```
직접 경쟁: [같은 문제를 같은 방식으로 푸는 서비스]
간접 경쟁: [같은 문제를 다른 방식으로 푸는 것 — 엑셀, 수작업 포함]
우리 차별점: [경쟁사가 못 하거나 안 하는 것]
```

### 6. 포지셔닝
```
{서비스명}은 {타겟}을 위한 {카테고리}로,
{핵심 혜택}을 제공합니다.
{대체재}와 달리 {차별점}입니다.
```

### 7. 페르소나 (대표 고객 1~2명)
```
페르소나 A:
  이름: [가상]
  직업: 
  상황: 
  고통: 
  원하는 결과: 
  반론: "..."
```

### 8. 반론/거절 처리
```
"비싸지 않아?" → [답변]
"기존 툴로 충분한데?" → [답변]
"설치/전환 귀찮은데?" → [답변]
"보안은?" → [답변]
```

### 9. 고객 언어
```
고객이 쓰는 단어: [전문용어 X, 고객 표현 O]
  - "{고객이 쓰는 표현}" → 우리 용어: {내부 용어}
  
쓰지 말아야 할 단어: [기술용어, 업계 은어]
```

### 10. 브랜드 보이스
```
톤: [전문적/캐주얼/따뜻한/도발적]
말투: [존댓말/반말/혼합]
금지 표현: [...]
참고 브랜드: [톤이 비슷한 브랜드]

예시 문장:
  좋은 예: "..."
  나쁜 예: "..."
```

### 11. 증거/신뢰 자산
```
수치: [사용자 수, 처리 건수, 절약 시간 등]
후기: [고객 추천사 요약]
인증: [보안 인증, 수상 등]
미디어: [소개된 매체]
```

### 12. 현재 목표
```
이번 달 목표: 
핵심 KPI: 
집중 채널: 
주의사항: 
```

## 에이전트별 참조 방법

| 에이전트 | 주로 보는 섹션 |
|----------|---------------|
| 카피라이터 | 3(고통), 4(혜택), 9(고객언어), 10(보이스) |
| SEO 담당 | 2(타겟), 5(경쟁), 9(고객언어) |
| SNS 담당 | 3(고통), 7(페르소나), 10(보이스) |
| 영업 | 3(고통), 6(포지셔닝), 8(반론처리), 11(증거) |
| CRO | 4(혜택), 5(경쟁), 8(반론처리) |

## 기존 시스템 연결

- `marketing_config.json` → 채널 설정 (frequency, tone, hashtags)
- `marketing_planner.py` → 이 컨텍스트를 자동 생성/업데이트
- `context_loader.py` → 에이전트 실행 시 자동 주입
- `ops_templates/marketing/` → 채널별 실행 템플릿 참조
