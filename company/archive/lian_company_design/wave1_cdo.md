# CDO 분석 — 리안 컴퍼니 Python CLI

## 디자인 비전
"비개발자 CEO가 터미널에서 혼자 생각하는 느낌" — 기다림이 지루하지 않고, 진행 상황이 명확하며, 문제가 생겨도 패닉하지 않는 CLI 경험

## UX 원칙

### 1. 친절한 전문가 원칙
- 리안은 비개발자. 에러 메시지에 `API Error 401` 같은 코드 대신 "API 키가 없거나 만료됐어. .env 파일 확인해봐."
- 모든 단계에서 "지금 뭐하는 중"을 보여줘야 함
- 빈 상태, 로딩, 에러 전부 리안이 이해할 수 있는 언어로

### 2. 진행 상황 = 신뢰
- 9개 에이전트가 순차 실행되는데 "얼마나 남았지?" 불안감 제거
- 각 에이전트 시작 시 헤더로 명확히 표시
- 전체 진행률 표시 (예: `[3/9] 하은 — 검증 중...`)

### 3. 개입 포인트는 딱 2곳
- 시은과의 명확화 대화 (최대 2번)
- 준혁의 NO-GO 판단 시 (여기서 멈추고 다시 생각할 기회)
- 나머지는 자동. 리안이 지켜보기만 해도 괜찮은 흐름

## 터미널 출력 포맷 설계

### 1단계: 배너 (첫 화면)
```
======================================================
          리안 컴퍼니 (LIAN COMPANY)
    아이디어 -> 설계서 자동 완성 AI 시스템
======================================================

💡 아이디어를 입력해줘 (엔터로 제출):
리안:
```

**의도**: 간결하고 친근. 리안이 "뭘 해야 하지?" 고민 없이 바로 입력.

---

### 2단계: 시은 명확화
```
============================================================
🤖 시은 | 오케스트레이터
============================================================
[스트리밍 응답...]

리안:
```

**의도**:
- `🤖 시은`으로 누가 말하는지 명확
- `| 오케스트레이터`로 역할 표시 (리안이 나중에 outputs 파일 볼 때 매칭 가능)
- 60자 구분선 = 시각적 호흡

---

### 3단계: 파이프라인 시작 전 확인
```

============================================================
리안:
```

**의도**: 마지막 확인. 리안이 "잠깐, 다시 생각해보니..." 할 기회.

### 5단계: NO-GO 케이스

```
============================================================
[5/8] ⚖️ 준혁 — 최종 판단 (Claude Opus)
============================================================
[스트리밍 응답...]

[판단 결과]
❌ NO-GO (종합 점수: 38/100)

사유: 시장 경쟁이 너무 치열하고 차별화 포인트가 약함.
수익모델도 불명확함.

============================================================

실행팀 진행 안 함. 아이디어를 수정하거나 다른 방향을 시도해봐.

📁 결과 저장: outputs/2026-03-18_143022_소상공인_인스타_캡션_자동화/
```

**의도**:
- `❌ NO-GO` = 실패가 아니라 "지금은 아니야"라는 판단
- 사유를 명확히 보여줘서 리안이 다음 시도에 반영 가능
- 저장된 파일로 나중에 다시 검토 가능

---

### 6단계: 조건부 GO 케이스

```
============================================================
[5/8] ⚖️ 준혁 — 최종 판단 (Claude Opus)
============================================================
[스트리밍 응답...]

[판단 결과]
⚠️ CONDITIONAL-GO (종합 점수: 58/100)

조건: 타겟을 좁히고(예: 뷰티 숍만), MVP는 인스타만 지원.
카카오톡 연동은 Phase 2로.

============================================================

조건을 확인하고 계속 진행할게.


============================================================
✅ 이사팀 완료. 실행팀 시작.
============================================================
```

**의도**:
- `⚠️ CONDITIONAL-GO` = 경고이지만 진행 가능
- 조건을 명시해서 실행팀(종범)이 구현 범위 조정
- 리안은 개입 없이 자동 진행

## 에러 메시지 UX

### 케이스 1: API 키 없음
```
❌ .env 파일에 ANTHROPIC_API_KEY가 없어.
   .env.example 참고해서 만들어줘.

   예시:
   ANTHROPIC_API_KEY=sk-ant-api03-...

   키는 새폴더/api.txt에 있어.
```

**의도**:
- 에러 코드 없음
- 해결 방법까지 바로 제시
- 리안이 어디서 키를 찾는지까지 알려줌

---

### 케이스 2: 네트워크 오류 (Perplexity API 실패)

```
============================================================
[2/8] 📊 서윤 — 시장 조사 (Perplexity 웹 검색 중...)
============================================================

❌ Perplexity API 연결 실패

가능한 원인:
  1. 인터넷 연결 끊김 → Wi-Fi 확인
  2. API 키 만료 → .env의 PERPLEXITY_API_KEY 재확인
  3. Perplexity 서버 이슈 → 잠시 후 재시도

지금까지 저장된 파일:
  ✓ 08_트렌드_태호.md
  ✓ 01_시장조사_서윤.md (일부)

다시 시도할까? [재시도 / 취소]
리안:
```

**의도**:
- 에러 원인 3가지 제시 (리안이 직접 판단 가능)
- 지금까지 진행된 게 날아가지 않았다는 안심
- 재시도 옵션 제공 (자동 재시도는 무한루프 위험)

### 케이스 3: 빈 응답 (AI가 아무것도 안 돌려줌)

```
============================================================
[3/8] 💡 민수 — 전략 수립
============================================================

⚠️ 민수가 빈 응답을 보냈어.

가능한 원인:
  1. 시은의 명확화가 너무 짧아서 컨텍스트 부족
```

- 리안에게 나중에 확인하라고 알림
- 원인을 추측해서 다음에 개선 가능

## 컴포넌트 시스템 (터미널 UI)

### 1. 헤더 컴포넌트
```python
def print_header(step: int, total: int, name: str, role: str, note: str = ""):
    """
    예시:
    ============================================================
    [3/8] 💡 민수 — 전략 수립
    ============================================================
    """
    emoji_map = {
        "시은": "🤖", "태호": "🎯", "서윤": "📊",
        "민수": "💡", "하은": "🔍", "준혁": "⚖️",
        "지훈": "📝", "종범": "🛠️", "수아": "📣"
    }
    emoji = emoji_map.get(name, "👤")
    header = f"[{step}/{total}] {emoji} {name} — {role}"
    if note:
        header += f" ({note})"

    print("\n" + "="*60)
    print(header)
    print("="*60)
```

### 3. 판정 컴포넌트
```python
def print_verdict(verdict: str, score: int, reason: str = ""):
    """
    [판단 결과]
    ✓ GO (종합 점수: 72/100)
    """
    emoji = "✓" if verdict == "GO" else "⚠️" if verdict == "CONDITIONAL_GO" else "❌"
    label = verdict.replace("_", "-")

    print(f"\n[판단 결과]")
    print(f"{emoji} {label} (종합 점수: {score}/100)")

    if reason:
        print(f"\n{reason}")
    print()
```

```python
      1. 인터넷 연결 끊김 → Wi-Fi 확인
      2. API 키 만료 → .env 재확인

    다시 시도할까? [재시도 / 취소]
    """
    print(f"\n❌ {title}\n")

    if causes:
        print("가능한 원인:")
        for i, cause in enumerate(causes, 1):
            print(f"  {i}. {cause}")
        print()

    if recovery:
        print(recovery)
        print()
```
