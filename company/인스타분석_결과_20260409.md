### [2] rtk 툴체인으로 토큰 88.9% 절감 + AI 모델 비교

**포스팅 내용**
- VS Code + Claude 환경에서 rtk(runtime toolkit) 명령어 프리픽스 시스템을 붙여서 토큰을 극단적으로 절감한 실전 사례
- 총 15,720개 명령어 실행, Input 166.3M 토큰 → 절감 후 실제 소비 36.3M (130M 절감, 88.9%)
- 명령어별 절감률:
  - `rtk read` — 100% 절감 (파일 내용 캐싱)
  - `rtk curl` — 22~100% 절감 (API 응답 캐싱)
  - `rtk lint` — 99.7% 절감 (lint 결과 캐싱)
  - `rtk map` — 99.4% 절감
  - `rtk wrap` — 99% 절감
- **AI 모델 성능 비교표 (포스팅 원문)**:
  | 순위 | 모델 | 1M 토큰 가격 | 속도(tok/s) | 지능지수 |
  |------|------|------------|------------|---------|
  | 1 | Google Gemini 3.1 Pro Preview | $4.50 | 123 | 57 |
  | 2 | OpenAI GPT-5.4 (xhigh) | $5.63 | 70 | 57 |

**core-shell 적용 방법**
- 이사팀→영업팀→마케팅팀 파이프라인에서 동일 사업체 정보를 여러 에이전트가 반복 조회할 때, 첫 번째 호출 결과를 캐싱하고 후속 에이전트는 diff만 처리
- CRM 조회, PDF 템플릿 로드, 고객 데이터 fetch 등 반복 호출 작업에 캐싱 레이어 추가
- 비용 최적화 라우팅: 속도 중요 작업(DM 자동생성, 실시간 응답)은 Gemini 3.1 Pro, 정밀도 필요 작업은 Claude로 분기

### [3] 폴더 계층 구조로 토큰 95% 절감 (Knowledge Graph 방식)

**포스팅 내용**
- AI에게 전체 문서를 던지는 대신, 정보를 계층으로 분류하고 필요한 레이어만 선택적으로 주입하는 PKM 방식
- **폴더 구조 (포스팅 원문)**:
  ```
  raw/        ← 수집 원본
  wiki/       ← 정제된 참고 문서
  analyses/   ← 분석 결과물
  concepts/   ← 핵심 개념 추출
  entities/   ← 실체 정보 (사람/장소/이름 등)
  sources/    ← 출처
  ```
- Graph view로 개념 간 링크 생성 → 관련 노드만 pull하는 selective context 패턴
- 예: 전체 문서 10만 토큰짜리를 통째로 주입하지 않고, entities(500토큰) + concepts(300토큰)만 주입 → 95% 절감

**core-shell 적용 방법**
- 소상공인 고객 데이터를 4계층으로 구조화:
  - `raw` — 수집 원본 (네이버 플레이스 크롤링 데이터, 상담 내용)
  - `entities` — 상호명, 업종, 위치, 매출 규모
  - `analyses` — 진단 결과 (현재 문제점, 점수)
  - `concepts` — 적용 전략, 포지셔닝
- 각 에이전트가 전체 CRM 대신 필요한 레이어만 주입:
  - PDF 진단서 생성 시 → `analyses` + `concepts`만
  - DM 생성 시 → `entities` + `concepts`만
  - 이사팀 전략 판단 시 → `analyses` + `entities`만

- 브랜드와 스토리 전문가 5명의 사례 기반 접근
- 롱블랙 연재 콘텐츠로 스페셜 시리즈 형태

**core-shell 적용 방법**
- 마케팅팀 에이전트 콘텐츠 생성 프롬프트에 '이야기 자본' 프레임 삽입:
  - 소상공인 DM/인스타/블로그 생성 시 단순 정보 나열 대신 "사장님의 서사(창업 계기, 위기 극복, 고객 변화)"를 중심으로 구성
  - 스토리 프롬프트 패턴: 현재 상황 → 원인 → 변화 가능한 미래
- PDF 진단서도 동일하게 '문제점 나열' 대신 '현재 상황→원인→변화 가능한 미래' 서사 구조 적용
- Perplexity로 WSJ/HBR 등 권위 있는 매체의 최신 트렌드를 주기적으로 스캔하여 후킹용 헤드라인 자동 추출하는 기능 추가
