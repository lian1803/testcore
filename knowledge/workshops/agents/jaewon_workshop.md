## ── 세계 최고 리서처 워크샵 — 10년차 시장조사 마스터클래스 ──

### 시장 리서치 프레임워크

#### TAM / SAM / SOM — 3단계 시장 규모 계산

| 레이어 | 정의 | 계산 방법 |
|--------|------|----------|
| TAM (Total Addressable Market) | 이 제품을 쓸 수 있는 지구상의 모든 사람이 다 쓰면? | 업종 전체 매출 or 전체 잠재 고객 수 × ARPU |
| SAM (Serviceable Addressable Market) | 우리가 실제로 도달할 수 있는 세그먼트 | TAM × (우리 타겟 지역/언어/규모 필터) |
| SOM (Serviceable Obtainable Market) | 현실적으로 3년 안에 먹을 수 있는 몫 | SAM × 현실적 점유율 (보통 1~5%) |

**Top-Down 접근 (속도 우선)**
- Gartner / Forrester / IDC 보고서에서 업종 전체 시장 수치 가져옴
- 거기서 타겟 세그먼트 비율 곱해서 SAM 산출
- 장점: 빠름. 단점: 투자자 앞에서 "이 수치 어디서 났어요?"에 약함

**Bottom-Up 접근 (신뢰도 우선, VC가 선호)**
- ICP(이상적 고객 프로파일) 조건에 맞는 기업/사람 수를 직접 카운팅
- LinkedIn Sales Navigator, Apollo.io, 정부 사업체 통계로 모수 확인
- 공식: `잠재 고객 수 × 연간 계약 단가(ACV) = TAM`
- 예시: "소상공인 마케팅 SaaS" → 국세청 자료상 5인 이하 요식업 사업체 수 × 월 3만원 × 12 = TAM
- 두 방식의 결과가 15% 이내로 수렴하면 가정이 탄탄한 것

**믿을 수 있는 수치 출처 (한국 기준)**
- 통계청 KOSIS — 사업체 수, 매출 규모
- 중소벤처기업부 창업통계 — 창업/폐업 트렌드
- 한국인터넷진흥원(KISA) — 디지털 전환 관련
- 네이버 데이터랩 — 검색량 기반 수요 추정
- 글로벌: Statista, IBISWorld, Crunchbase (유료지만 유료만큼 신뢰)

### 경쟁사 인텔리전스 심화

#### 경쟁사 랜딩 분석 체계 — 4가지 레이어

**레이어 1: 카피 패턴 분석**
- Hero 섹션 H1 — 어떤 Pain을 건드리는가? 어떤 결과를 약속하는가?
- 서브헤드 — Feature 나열 vs. Benefit 중심 중 어느 쪽인가?
- CTA 버튼 텍스트 — "무료 체험", "지금 시작", "데모 신청" 중 어느 단계 타겟팅?
- 소셜 프루프 위치 — 히어로 바로 아래 vs. 중간 vs. 하단
- 긴급성/희소성 문구 존재 여부

**레이어 2: 포지셔닝 갭 발굴**
- 경쟝사 3곳 이상 Hero 카피를 나란히 놓으면 "아무도 안 말하는 것"이 보임
- 모두 "자동화"를 말하면 → "투명성/통제권"이 갭
- 모두 "빠름"을 말하면 → "정확도/신뢰"가 갭
- 갭 = 우리 포지셔닝 후보

**레이어 3: 가격 전략 해독**
- 공개 가격 vs. "문의하기" — 가격 미공개는 엔터프라이즈 타겟 신호
- 플랜 구조 — Freemium, Free Trial, Demo-first 중 어느 것?
- 가격 앵커링 — 가장 비싼 플랜이 먼저 나오는가?
- 연간/월간 토글 — 할인율 몇 %?

**레이어 4: 고객 불만 수집처**
| 플랫폼 | 사용법 | 특징 |
|--------|--------|------|
| G2.com | 경쟝사 이름 검색 → "Cons" 탭 집중 | B2B 소프트웨어 특화 |
| Capterra | 1~2점 리뷰만 필터 | SMB 사용자 많음 |
| Reddit | "r/[업종] + 경쟝사명" 검색 | 무편집 솔직함 |
| App Store / Play Store | 업데이트 후 리뷰 급증 구간 확인 | 기능 불만 집중 |
| Twitter/X | `"[경쟝사명] is" OR "[경쟝사명] hate"` | 실시간 감정 |
| 네이버 블로그/카페 | "후기 단점" 검색 | 국내 B2C에 강력 |

**툴 스택 (무료 중심)**
- Wayback Machine — 과거 랜딩 변천사 확인 (전략 방향 유추)
- SimilarWeb 무료 플랜 — 트래픽 추정, 유입 채널 비율
- SpyFu — 경쟝사 PPC 키워드 (어떤 Pain으로 광고 태우는지)
- Google Ads Transparency Center — 실제 집행 중인 광고 카피 확인

### Pain Point 발굴 기법

#### 인터뷰 없이 온라인에서 Pain 캐내는 7가지 방법

**1. Reddit 채굴 (가장 강력)**
- 검색: `site:reddit.com "[업종/직종] 힘들다" OR "문제" OR "불편"`
- 타겟 서브레딧 예시: r/smallbusiness, r/entrepreneur, r/마케팅 (한국어 커뮤니티)
- 업보트 수 많은 댓글 = 집단적으로 공감하는 Pain → 시장 검증된 신호
- 패턴 찾기: 50~100개 포스트 분석 후 반복 등장하는 단어/감정 태깅

**2. 경쟝사 리뷰 역공학**
- G2 / Capterra "단점(Cons)" 섹션 — 경쟝사의 미해결 Pain = 우리의 기회
- 1~2점 리뷰에서 직접 인용구 추출 → 나중에 카피에 "그 말 그대로" 활용

**3. 검색어 분석 (수요 = Pain의 외현)**
- 네이버 키워드 도구: "[타겟 직업] + 방법", "[문제상황] + 해결"
- Google Search Console (자체 사이트 있으면): 사람들이 실제로 뭘 검색해서 오는지
- "사람들이 묻는 것(People Also Ask)" 박스 — Pain의 언어 그대로

**4. 커뮤니티/카페 모니터링**
- 네이버 카페: 업종별 사장님 모임, 프리랜서 커뮤니티
- 오픈카톡방, 밴드 — 더 날것의 불만
- Facebook 그룹 — 업종별 그룹에서 "도움 요청" 포스트

**5. 유튜브 댓글 채굴**
- 경쟝사 제품 리뷰 영상 댓글 → "이게 안 돼요", "이것만 됐으면" 패턴
- "단점" 포함된 댓글만 필터 → Pain 목록 직접 생성

**6. Job 포스팅 역독(逆讀)**
- 경쟝사가 "마케팅 자동화 담당자 채용" → 이 기능을 내재화 못 했다는 신호
- 고객사가 "데이터 분석가 채용" → 현재 툴이 분석을 못 해준다는 신호

**7. 검색 자동완성**
- 구글/네이버에서 "[타겟] + ..." 입력 시 자동완성 — 실제 검색 빈도 순 정렬
- "[경쟝사명] + 대안" / "[경쟝사명] + 문제" 자동완성 → 이탈 이유 파악

### 리서치 결과 신뢰도 평가

#### "이 수치 믿어도 되나" — 5단계 판단 기준

**1단계: 출처 유형 확인**
| 신뢰도 | 출처 유형 | 주의사항 |
|--------|----------|---------|
| 높음 | 정부 통계, 학술 논문, 독립 리서치 기관 | 최신성 확인 필요 |
| 중간 | 유명 컨설팅사 공개 보고서(맥킨지, BCG) | 샘플 크기/방법론 확인 |
| 낮음 | 기업 자체 발행 "State of X" 보고서 | 자사 제품 홍보 의도 내재 |
| 주의 | PR 기사, 보도자료 | 팩트 선택적 인용 많음 |
| 불신 | 출처 불명 인포그래픽 | "연구에 따르면" 뒤에 출처 없으면 폐기 |

**2단계: 방법론 투명성**
- 샘플 크기 명시? (n=50과 n=5000은 다르다)
- 조사 기간 명시? (2019년 데이터를 2025년 트렌드로 쓰면 안 됨)
- 조사 대상 명시? (미국 기업 500명 대상 데이터를 한국 소상공인에게 적용하면 오류)

**3단계: 교차 검증**
- 독립된 2개 이상의 출처가 비슷한 수치 → 신뢰
- 출처 1개뿐 + 다른 곳에서 수치 다름 → 의심, 원출처 추적
- "이 수치 어디서 왔지?" 역추적 → 대부분 하나의 오래된 보고서에서 시작

**4단계: 최신성**
- SaaS / AI / 마케팅 업종: 2년 이상 된 데이터는 의미 감소
- 규제/법률: 최신 개정안 확인 필수
- 시장 규모 수치: 연도 표기 없으면 무조건 출처 확인

**5단계: 1차 vs 2차 자료 구분**
- **1차 자료 (Primary)**: 직접 수집. 인터뷰, 설문, 직접 관찰 — 최신, 목적 맞춤, 비용 높음
- **2차 자료 (Secondary)**: 기존 자료 재활용. 보고서, 논문, 통계 — 빠름, 비용 낮음, 오래됨
- 베스트 프랙티스: 2차 자료로 가설 수립 → 1차 자료(소규모라도)로 검증
- 둘 다 같은 방향 가리키면 → 확신 가져도 됨

### 실시간 트렌드 포착

#### Perplexity + Google Trends + X(트위터) 3중 조합법

**Step 1: Google Trends — 수요 방향 확인 (무료, 즉시)**
- 키워드 입력 → "관심도 시간 흐름" 그래프 — 상승/하락/계절성 파악
- "관련 검색어 → 급상승" 탭 — 아직 작지만 빠르게 커지는 신호
- 비교 기능: 경쟝사명 vs. 우리 카테고리 키워드 — 상대적 관심도
- 지역 필터: 한국 내 지역별 관심도 차이 확인 (서울 vs. 지방)

**Step 2: Perplexity — 맥락 + 최신 정보 (핵심 도구)**
- "지금 [업종]에서 가장 많이 거론되는 문제가 뭔가?"
- "최근 3개월 [키워드] 관련 뉴스와 변화는?"
- "한국에서 [타겟] 사이에서 요즘 뜨는 키워드는?"
- Perplexity는 실시간 인터넷을 읽으므로 리포트보다 6개월 빠른 정보 가능

**Step 3: X(트위터) — 조기 신호 포착**
- Advanced Search: `"[키워드]" lang:ko since:2025-01-01` — 최근 언급 트렌드
- 감정 분석: 같은 키워드도 "드디어 해결됨" vs. "아직도 이게 문제" 맥락 다름
- 인플루언서 + 일반인 반응 구분 — 인플루언서 = 트렌드 증폭, 일반인 = 실제 수요

**조합 시너지 공식**
```
Google Trends 상승 키워드
    + Perplexity "이게 왜 뜨나?" 맥락 파악
    + X에서 실제 감정/언어 확인
= 리얼 타임 시장 신호 + 그걸 표현하는 고객 언어
```

**한국 특화 트렌드 소스 추가**
- 네이버 데이터랩 쇼핑인사이트 — 실구매 연결 검색어
- 카카오 채널 트렌드 — 구독자 반응 실시간
- 오픈카톡 + 네이버 카페 — 미디어 이전에 퍼지는 풀뿌리 신호

```markdown
## Executive Summary (30초 요약)
[이 리서치의 핵심 결론 2~3문장. 읽는 사람이 전체 안 읽어도 의사결정 가능한 수준]

## 핵심 발견 3가지
1. **[발견 1]** — 출처: [URL] | 신뢰도: 높음/중간/낮음
   - 근거: [구체적 수치 or 인용]
   - 임플리케이션: [이게 우리에게 의미하는 것]

2. **[발견 2]** — 출처: [URL] | 신뢰도: 높음/중간/낮음
   - 근거: [구체적 수치 or 인용]
   - 임플리케이션: [이게 우리에게 의미하는 것]

3. **[발견 3]** — 출처: [URL] | 신뢰도: 높음/중간/낮음
   - 근거: [구체적 수치 or 인용]
   - 임플리케이션: [이게 우리에게 의미하는 것]

## 승현(전략가)에게 — 전략 임플리케이션
- [발견 1이 시사하는 포지셔닝 방향]
- [경쟝사 갭에서 나온 차별화 포인트]
- [시장 진입 타이밍 판단 근거]

## 예진(카피라이터)에게 — 언어 인사이트

```

**전달 규칙**
- 숫자는 반드시 출처 URL 병기. "업계 전문가에 따르면" 금지.
- 신뢰도 낮은 데이터는 명시. 숨기면 전략 오판으로 이어짐.
- 보고서 길이: Executive Summary 충실하게, 세부 데이터는 부록. 승현/예진이 바쁘다.
- 불확실한 것은 "불확실함"이라고 말하는 게 틀린 수치 내놓는 것보다 낫다.
