## ── 세계 최고 Analytics 워크샵 — 10년차 그로스 애널리스트 마스터클래스 ──

### 📊 SaaS 핵심 지표 체계 (North Star + AARRR)

**North Star Metric(NSM) 이란**
단 하나의 숫자로 "우리 제품이 고객에게 진짜 가치를 주고 있는가"를 나타내는 지표.
매출이나 DAU 같은 결과 지표가 아니라, 가치 전달 행동 지표여야 한다.

| 제품 유형 | North Star 예시 |
|-----------|----------------|
| 협업 SaaS | 주간 활성 팀 수 (2명 이상 협업한 팀) |
| 분석 SaaS | 의사결정자가 실제로 조회한 대시보드 수 |
| 커머스 SaaS | 반복 구매한 셀러 수 |
| 에듀테크 | 강의 완강 후 과제 제출한 수강생 수 |

NSM은 제품이 진화하면 함께 바꿔야 한다. 고정된 것이 아니다.
초기 설계 기간: 2~4주면 충분. 첫 분기 안에 조직 전체가 이걸 보도록 세팅해라.

**AARRR — SaaS에서의 정의 + 벤치마크**

| 단계 | 정의 | SaaS 핵심 지표 | 업계 벤치마크 |
|------|------|--------------|-------------|
| Acquisition (획득) | 처음 유입되는 경로 | 채널별 방문자, CAC | 방문→회원가입 전환 2~5% |
| Activation (활성화) | "아, 이게 되네" 첫 경험 | Aha Moment 도달율 | 신규 유저 40~60% 달성이 목표 |
| Retention (리텐션) | 다시 돌아오는가 | D1/D7/D30, MRR 유지율 | D30 평균 6% (앱 전체), B2B SaaS 월 이탈 3~8% |
| Referral (추천) | 스스로 퍼트리는가 | NPS, 바이럴 계수(K) | K > 1 이면 바이럴 성장 |
| Revenue (수익) | 돈을 내는가 | MRR, ARPU, LTV:CAC | LTV:CAC 3:1 이상이 건강한 SaaS |

**바닐라 지표를 피하는 법**
- 페이지뷰 → 버려라. "핵심 기능 사용 횟수"로 교체
- 전체 가입자 수 → "활성 유저(7일 내 핵심 행동)" 로 교체
- 앱 설치 수 → D7 리텐션으로 교체

### 🔍 퍼널 분석 심화

**이탈 구간 정확히 찾는 3단계**

1. **전체 퍼널 그리기** — 랜딩 → 회원가입 → 온보딩 → 핵심 기능 첫 사용 → 유료 전환
   각 단계 사이 전환율 계산. 전환율이 가장 낮은 구간 = 1차 타깃
2. **이탈 유형 분류**
   - 기술적 이탈: 버그, 로딩 속도, 모바일 깨짐 → BE/FE 수정으로 즉시 해결 가능
   - 동기 이탈: 가치를 못 느끼고 떠남 → 카피, 온보딩, Aha Moment 재설계
   - 마찰 이탈: 단계가 너무 많음 → UX 간소화, 폼 필드 줄이기
3. **세그먼트 분리 분석** — 전체 평균은 거짓말한다. 채널별/기기별/플랜별로 쪼개라

**우선순위 결정 프레임워크 (ICE Score)**

```
ICE = Impact × Confidence × Ease (각 1~10점)
```

| 개선 항목 | Impact | Confidence | Ease | ICE Score |
|-----------|--------|-----------|------|-----------|
| CTA 문구 변경 | 6 | 8 | 9 | 432 |
| 온보딩 단계 축소 | 9 | 7 | 5 | 315 |
| 가격 페이지 개편 | 8 | 5 | 3 | 120 |

ICE가 높은 것부터 실행. 단, Confidence가 3 이하면 먼저 데이터 더 모아라.

**2025 SaaS 퍼널 병목 통계**
- MQL→SQL 전환이 평균 15~21% (가장 큰 병목)
- 이 구간 5포인트 개선 = 매출 최대 18% 상승
- 가격 페이지 명확성이 있는 SaaS는 전환율 11% 높음
- 라이브 채팅 추가 시 데모 요청 20% 증가

**GA4 퍼널 분석 실전 세팅**
- Explore → Funnel Exploration → 단계 정의
- Open Funnel(비선형 여정) vs Closed Funnel(순서 강제) 목적에 맞게 선택
- 핵심 지표: 단계별 유저 수, 전환율, 이탈율, 단계 간 경과 시간
- 세그먼트 적용: 채널별, 기기별로 비교해 이탈 패턴 차이 파악

### 🧪 A/B 테스트 설계 원칙

**테스트 전 반드시 정하는 3가지**
1. **가설**: "X를 Y로 바꾸면 Z가 W% 개선될 것이다" — 구체적으로 적어야 한다
2. **성공 지표**: 무엇이 몇 % 변하면 승리인가 (주 지표 1개 + 보조 지표 2개 이하)
3. **샘플 크기와 기간**: 계산 없이 시작하면 무효

**샘플 크기 계산 공식**

```
필요 변수:
- 기준 전환율 (현재 CVR) — 예: 3%
- MDE (최소 감지 효과) — 예: 10% 상대 개선 = 3% → 3.3%
- 유의수준 (α) = 0.05 (95% 신뢰도)
- 검정력 (β) = 0.80 (80% 파워)

결과: 변형당 최소 ~30,000 방문자, 3,000+ 전환 필요
도구: statsig.com/calculator, optimizely sample size calculator
```

**테스트 기간**
- 최소 2주, 최대 6~8주
- 요일 효과 반드시 포함 (주중 vs 주말 패턴)
- 샘플 크기 달성 전에 중단하지 마라 — 거짓 양성(False Positive) 위험

**무엇을 테스트할 것인가**

테스트해야 하는 것 (임팩트 큰 순):
1. 가격 페이지 구조 + CTA 문구
2. 온보딩 첫 화면 (Aha Moment 경로)
3. 이메일 제목 + 발송 타이밍
4. 체크아웃/결제 플로우 단계 수
5. 소셜 프루프 위치/형태

테스트하지 말 것:
- 트래픽이 월 1만 이하인 페이지 (통계적으로 무의미)
- 동시에 2개 이상 변수 변경 (원인 특정 불가)
- 이미 명확한 UX 버그 (고치고 나서 테스트)

**통계적 유의성 vs 실질적 유의성**
- p < 0.05 달성해도 전환율 0.1% 개선은 의미 없다
- "이 결과가 비즈니스에 실질적 임팩트가 있는가"를 항상 함께 판단

### 📈 코호트 분석 + 리텐션 곡선

**D1/D7/D30 벤치마크 (2025 기준)**

| 구분 | D1 | D7 | D30 |
|------|----|----|-----|
| 전체 앱 평균 | 25~35% | 10~15% | 6% |
| 우수 B2B SaaS | 60%+ | 40%+ | 25%+ |
| 소비자 구독 | 33% | 12% | 6~10% |

D30이 6%라는 것은 가입자 94%가 한 달 안에 이탈한다는 뜻. 이게 업계 평균이다.
이 숫자를 10%로 올리는 것이 CAC 낮추는 것보다 보통 ROI가 높다.

**코호트 분석 실전 방법**

```
코호트 구성 기준:
- 가입 시기별 (월별 코호트 — 가장 기본)
- 획득 채널별 (유기 검색 vs 유료 광고 vs 레퍼럴)
- 플랜 유형별 (무료 vs 트라이얼 vs 유료)
- 첫 번째 행동별 (Aha Moment 달성 vs 미달성)
```

**리텐션 곡선 해석법**

- 곡선이 0에 수렴: 제품-시장 적합(PMF) 미달. 코어 가치부터 다시 봐라
- 곡선이 일정 수준에서 수평: PMF 달성. 수평 지점의 높이가 핵심
- 초기 급락 후 수평 안정: 헤비 유저 세그먼트가 있음 → 이 사람들을 분석해라

**GA4 코호트 Exploration 세팅**
- Explore → Cohort Exploration
- Cohort Inclusion: 첫 방문(First Touch) 또는 특정 이벤트 완료
- Granularity: 일별(D1~D7 측정) 또는 주별(W1~W12)
- Return Criteria: 핵심 기능 재사용 이벤트로 설정 (세션 수가 아니라 의미 있는 행동)

**월별 코호트 리텐션 테이블 예시**

| 가입월 | M0 | M1 | M2 | M3 | M6 |
|--------|----|----|----|----|-----|
| 2025-01 | 100% | 42% | 31% | 28% | 22% |
| 2025-02 | 100% | 38% | 27% | 24% | — |
| 2025-03 | 100% | 45% | 33% | — | — |

M3 이후 안정화되면 그게 진짜 리텐션. M1 급락은 온보딩 문제.

**B2B SaaS 월간 이탈율 벤치마크 (2025)**
- 엔터프라이즈(ASP $500+): 월 3.2% 이하가 건강
- SMB: 월 5~7%
- 소비자 구독: 월 7.8%
- 첫 3개월 유지율: 고가 플랜 98%, 저가 플랜($10 이하) 87%

### 💡 인사이트 → 액션 변환

**데이터로 끝내지 않는 5단계**

1. **관찰** (What): "3단계 이탈율이 지난주 대비 12% 증가했다"
2. **진단** (Why): "모바일에서만 이탈 증가 + 신규 버전 배포일과 겹침"
3. **가설** (So What): "v2.3 업데이트의 폼 UI 변경이 모바일 이탈 원인"
4. **액션** (Now What): "FE에 모바일 폼 롤백 요청, 병렬로 Hotjar 세션 리플레이 확인"
5. **측정** (Then What): "배포 후 48시간 내 모바일 이탈율 기준점 복귀 여부 확인"

**FE/BE 지시서 작성 원칙**
- 나쁜 예: "전환율 개선해주세요"
- 좋은 예: "회원가입 3단계 → 2단계로 축소. 생년월일 필드 제거. 이메일+비밀번호만 남김. 이유: 3단계 이탈 68%, 대부분 생년월일 입력 후 이탈(Hotjar 확인)"

**임팩트 추정법**
```
예상 추가 전환 = (현재 트래픽) × (현재 CVR) × (예상 개선율)
예시: 월 10,000 방문 × 3% CVR × 20% 개선 = +60건/월
ARPU $50 기준 = $3,000/월 추가 수익 예상
```

**우선순위 결정 원칙**
1. 이탈 볼륨 × 개선 가능성 = 실질 임팩트
2. 빠른 실험(1일 구현) 먼저, 대규모 개편은 증거 쌓은 뒤
3. 한 번에 하나씩 — 동시 다발 변경은 원인 추적 불가

### 🔄 스프린트 루프 설계

**1주 분석 → 가설 → 실험 → 측정 사이클**

```
월요일 (분석)
- 지난 주 핵심 지표 검토 (North Star + AARRR 각 단계)
- 이상값 탐지: 전주 대비 ±10% 이상 변동 구간 플래그
- 코호트 리텐션 업데이트
- 이번 주 분석 타깃 1개 결정

화요일 (진단)
- 플래그 구간 심층 분석 (세그먼트 분리, GA4 Exploration)
- Hotjar / 세션 리플레이 / 사용자 인터뷰 데이터 교차 확인
- 근본 원인 3가지 가설 작성

수요일 (우선순위 + 설계)
- ICE Score로 가설 순위 결정
- 1순위 가설 → A/B 테스트 설계 OR 즉시 수정 결정
- 샘플 크기 계산 (테스트라면)
- FE/BE 지시서 작성

목~금 (실행)
- 즉시 수정 건: FE/BE 배포
- 테스트 건: 세팅 완료 + 런칭
- 다음 주 측정 기준점 문서화

다음 월요일
- 결과 측정 → 루프 반복
```

**측정 기준 문서 템플릿**

```markdown
## 실험 #[N] — [실험명]
- 가설: [구체적 가설]
- 변경 내용: [FE/BE 정확히 무엇을 바꿨나]
- 성공 기준: [지표 X가 Y% 이상 개선]
- 측정 기간: [날짜 범위]
- 최소 샘플: [계산값]
- 담당: [FE/BE 이름]
- 결과: [미기입 → 측정 중]
```

**장기 성장 루프 설계 (6주 이상)**

| 주차 | 포커스 | 기대 효과 |
|------|--------|----------|
| 1~2 | 획득 채널 전환율 최적화 | 신규 유입 품질 향상 |
| 3~4 | 온보딩 + Aha Moment 개선 | Activation율 +10~20% |
| 5~6 | 리텐션 트리거 이메일 + 인앱 넛지 | D7 리텐션 +5% |
| 7~8 | 유료 전환 장벽 제거 | Trial → Paid +3~5%p |
| 9~10 | LTV 확장 (업셀/크로스셀) | ARPU 상승 |
| 11~12 | 레퍼럴 루프 설계 | K-factor 측정 시작 |

**스프린트 회고 체크리스트**
- [ ] 이번 주 가설 중 맞은 것 / 틀린 것 비율
- [ ] 틀린 가설 — 왜 틀렸나? 데이터 부족? 세그먼트 착오?
- [ ] 다음 스프린트에서 더 잘하려면 무엇을 바꿀 것인가
- [ ] 실험 문서 업데이트 완료 여부
