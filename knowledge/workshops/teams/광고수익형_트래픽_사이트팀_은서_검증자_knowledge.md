# Google AdSense 정책 위반 및 계정 정지 원인 (2024-2025)

## 핵심 수치
**2024년 Google은 전 세계적으로 1,270만 개 광고주 계정을 정지했으며, 55억 개 이상의 광고를 삭제 또는 제한했습니다.**[6] 2025년에도 이러한 추세가 계속되고 있습니다.

## 즉시 정지 대상 중대 위반 (1회 = 1 스트라이크)

### 1. 시스템 우회 (Circumventing Systems)
가장 심각한 위반으로 광고 검토 프로세스를 우회하려는 모든 시도를 포함합니다[2]:
- **클로킹**: 사용자와 Google에 다른 콘텐츠 표시[2]
- **광고 성분 조작**: 제한된 용어를 의도적으로 오타내기[2]
- **다중 계정 생성**: 이전 정지를 우회하기 위해 새 계정 개설[2]
- **숨겨진 리다이렉트**: 기만적인 랜딩 페이지 사용[2]

**결과**: 경고 없이 즉시 계정 정지, 영구 광고 플랫폼 추방[3]

### 2. 허위 표현 (Misrepresentation)
**일반 위반 5회 = 1 스트라이크 | 중대 위반 = 즉시 1 스트라이크**[1]

**일반 수준**[1]:
- "더 나은 신용 점수를 받을 자격이 있습니다"(방법 미명시)
- "오늘 최고의 항공편 거래를 찾으세요"(실제 목록 없음)

**중대 위반**[1]:
- "무료 iPhone을 받으세요"(허위)
- "Amazon이 당신 근처에서 운전자 고용"(거짓)
- "하버드 입학 허가를 받으세요"(기만)

**금지 행위**[2]:
- 성과, 결과, 가격에 대한 거짓 주장
- 수수료 또는 약관 숨김
- 합법적인 브랜드 사칭

### 3. 네트워크 남용 (Ad Network Abuse)
- **클릭 사기**: 봇으로 노출/클릭 인위적 증가[2]
- **무효 트래픽**: 기만적인 캠페인 행동
- **결과**: 장기 계정 정지[2]

### 4. 부적절한 콘텐츠[2]
- 혐오, 폭력, 성인 콘텐츠, 차별 홍보
- 자극적이거나 공포 기반 메시지
- 정치, 건강, 금융 같은 민감한 분야의 경계선 메시지

### 5. 악성코드 및 불법 소프트웨어[4]
- 스파이웨어, 친밀한 파트너 감시 기술[5]
- GPS 트래커(무단 추적 목적)
- 불법 감시 장비 홍보[5]

## RSOC(Refined Ads On Content) 

### Google helpful content update penalty signs how to detect and recover from algorithmic penalty for AI content sites
### **Google Helpful Content Update Penalty 징후 탐지**
AI 콘텐츠 사이트는 **site-wide 신호**로 전체 트래픽 5-18% 하락(2022-2025 Semrush 데이터), 특히 **경쟁 토픽의 generic how-to 콘텐츠**에서 60% 손실 발생[3][2].  
**주요 징후 (Search Console + GA4 확인)**:
- GSC에 **manual action 알림 없음** → 알고리즘 페널티 (core update 타이밍 맞물림)[3].
- **경쟁 키워드 랭킹 급락**: "how to X" 형식, AI-like 연구 기반 텍스트 (경험 부족)[2][1].
- **트래픽 패턴**: informational 페이지 중심, AI 생성 thin/generic 콘텐츠 (스크린샷/케이스스터디 미포함)에서 집중 하락[2][5].
- **영향 범위**: 사이트 전체 (2022 HCU site-wide), 2024 March Core 후 연속 모니터링[1][5].
- **AI 사이트 특성**: "experience dilution" – 정확하지만 **first-hand expertise 없음** (gotchas, 실패 사례 미포함)[2].

| 징후 유형 | 구체 사례 | 확인 툴 |
|-----------|----------|---------|
| **트래픽 드롭** | 60% 손실 (기본 튜토리얼)[2] | GA4 (organic only) |
| **랭킹 변화** | Page 1 → Page 3+ (뉴스 요약/AI 블로그)[2][4] | GSC Performance |
| **콘텐츠 특징** | Generic, no original data/screenshots[2] | 수동 audit (E-E-A-T 체크) |

### **탐지 프레임워크 (3단계, 1주 내 실행)**
1. **GSC 검토**: Manual Actions 탭 확인 → 없으면 알고리즘[3][6].
2. **상관 분석**: Google update 타임라인 (e.g., Dec 2025 HCU) vs 트래픽 드롭 날짜[2][5].
3. **콘텐츠 스코어링**: 100페이지 샘플 → "연구 vs 경험" 비율 측정 (예: 스크린샷 0개 = 위험)[2].

### **회복 전략 (실전 로드맵, 3-6개월 효과)**
**핵심: unhelpful 콘텐츠 제거 + E-E-A-T 강화**. Classifier가 장기 모니터링 → 개선 시 신호 해제 (수개월 소요)[1].  
**작동 확인 사례**: 이전 HCU 피해 사이트, 전문성 추가 후 회복[2].

| **DO (작동)** | **프레임워크/수치** | **예시** |
|---------------|---------------------|----------|
| **전문성 증명** | First-hand: gotchas + 실패 사례 추가 (20% 콘텐츠 비중)[2] | "이 방법 실패한 3가지 + workaround" 섹션 |
| **구체화** | Screenshots/examples 5+ per post, original data[2] | 코드 스니펫 + "내 테스트 결과: 30% 속도 향상" |
| **유지 업데이트** | 분기별: 신규 개발 + "생각 변화" 섹션[2] | Date update + "2026 변화: Y 대신 Z 사용" |
| **제한 명시** | Trade-offs/비적합 케이스 (신뢰 ↑)[2] | "이 방법 초보자 X, 고급자 O" |
| **저품질 제거** | Thin 페이지 30% 삭제 (historical authority 유지)[1][2] | Noindex or 301 to hub page |

| **DON'T (실패)** | **이유/손실 사례** |
|-------------------|---------------------|
| **단순 길이 늘리기** | 1.5k→3k fluff → 추가 랭킹 ↓[2] |
| **Fake AI expertise** | Fabricated 케이스 탐지 → 악화[2] |
| **삭제 과다** | Authority 상실 → 회복 지연[2] |
| **Date만 변경** | No real update → 무효[2] |

**AI 사이트 특화**: **human rewrite 70% + 전문가 기고** (e.g., 10 포스트에 2명 컨설턴트).
