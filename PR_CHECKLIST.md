# PR 체크리스트

작업 올리기 전에 아래 항목 확인. Claude Code에 "PR 체크해줘" 라고 하면 자동으로 봐줌.

---

## 필수 확인 (전부 YES여야 올릴 수 있음)

### 기본
- [ ] `git pull` 하고 최신 코드 기준으로 작업했음
- [ ] 내 컴터에서 실제로 돌아가는 거 확인했음
- [ ] 하드코딩된 개인 경로 없음 (`C:/Users/본인이름/...` 같은 거)

### 디자인 (UI 수정한 경우)
- [ ] 모바일에서도 깨지지 않음 (브라우저 창 줄여서 확인)
- [ ] 폰트 크기 계층 유지됨 (제목 > 소제목 > 본문)
- [ ] 색상이 기존 팔레트에서 벗어나지 않음

### 코드
- [ ] 다른 팀원 기존 코드 지우거나 덮어쓴 거 없음
- [ ] API 키나 비밀번호를 코드에 직접 넣지 않음

---

## 이런 거 하면 안 됨

- `main`에 직접 push → `git push core-shell main` 하기 전에 리안한테 먼저 말하기
- `company/venv/`, `*.pyc`, `__pycache__/` 올리기 → `.gitignore`가 막아줌
- `.env` 파일 올리기 → 절대 금지 (API 키 유출)

---

## 잘 모르겠으면

Claude Code에 이렇게 말해:

```
내가 수정한 파일들 PR 체크해줘
```

Claude가 위 항목들 기준으로 자동으로 확인해줌.
