# 팀 협업 규칙 (깃 사용법)

개발자 아니어도 됨. 딱 3가지만 기억해.

---

## 왜 `git pull`을 먼저 해야 하냐

**비유**: 우리 팀이 구글독스 한 장을 같이 편집하고 있어.  
혜경누나가 어젯밤에 3줄 추가했어. 근데 내가 그걸 모르고 옛날 버전 위에 내용을 쓰면?  
→ 혜경누나 작업이 날아가거나 충돌 남.

`git pull` = **"혹시 남이 새로 쓴 거 있어? 먼저 가져올게."**

**규칙: 작업 시작 전 무조건 먼저:**
```bash
git pull
```

---

## 내 작업이 남 작업을 지우지 않으려면

**비유**: 기차 칸 연결하기.  
혜경-1칸 → 혜경-2칸 → 리안-3칸 → 태은-4칸  
이렇게 뒤에 내 칸을 붙이면 아무도 안 지워짐.

근데 내가 pull 안 하고 올리면 기차 중간에 끼어들려다가 탈선 남.

**작업 순서:**
```
1. git pull          ← 최신 가져오기
2. 작업하기
3. git push          ← 내 작업 올리기
```

---

## 충돌(Merge Conflict)이 났을 때

**비유**: 혜경이랑 나랑 같은 줄에 동시에 다른 내용 적었을 때.  
깃이 "야 둘 다 같은 줄 고쳤는데 어떻게 할까?" 하고 알려주는 것.

터미널에 이런 게 뜸:
```
<<<<<<< HEAD
내가 쓴 내용
=======
혜경이가 쓴 내용
>>>>>>> 
```

**해결법:**
1. 리안한테 연락 ("충돌났어요")
2. 아니면 Claude Code에 "충돌 해결해줘" 라고 말하기

혼자 해결하려다 잘못되면 더 꼬임. 그냥 리안한테 말하는 게 제일 빠름.

|------|----------|

| 뭔가 이상하면 | 리안한테 연락 |
