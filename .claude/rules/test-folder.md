# 테스트 파일 저장 규칙

모든 테스트/실험/임시 HTML, 스크립트, 프로토타입은 반드시 아래 서브폴더에 분류:

```
test/
├── 디자인/   — HTML 랜딩/UI 실험, 쉐이더 테스트, 애니메이션 프로토타입
├── 마케팅/   — 카피 테스트, 이미지 소재, 메시지 실험
├── 영업/     — 영업 스크립트 실험, CRM 테스트, 멘트 초안
├── 개발/     — 파이썬 스크립트, API 테스트, 기능 검증 코드
└── 기타/     — 위 분류에 안 맞는 것
```

## 절대 금지
- 루트(`core/`)에 테스트 파일 생성
- `team/`, `company/` 안에 test_ 접두사 파일 생성
- `test/` 바로 아래(서브폴더 없이) 파일 생성

## 파일명 권장
```
test/디자인/YYYY-MM-DD_설명.html
test/개발/YYYY-MM-DD_설명.py
```
