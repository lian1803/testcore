# 코딩 작업 시 서브에이전트 기준

| 작업 규모 | 행동 |
|-----------|------|
| 1개 파일, 30줄 이하 수정 | Claude 직접 처리 |
| 새 모듈/클래스 (100줄+) | `be` 에이전트 스폰 |
| 프론트엔드 컴포넌트/UI | `fe` 에이전트 스폰 |
| 아키텍처 설계 필요 | `cto` → `be`/`fe` 순서 |
| 새 팀/에이전트 시스템 | `build_team.py` (교육팀) |
| 테스트/QA 필요 | `qa` 에이전트 스폰 |

스폰: `Agent(subagent_type="be", prompt="...")`

## Worktree + Batch 자동 적용 (리안이 말 안 해도 항상)

**Worktree** (`isolation: "worktree"`) 적용:
- 여러 서브에이전트 동시 스폰 시
- be/fe/qa가 같은 파일 건드릴 가능성 있을 때
- 기존 코드에 영향 줄 수 있는 큰 변경

**Batch** (단일 메시지에 여러 Agent 동시 호출) 적용:
- 독립 작업 2개 이상이 동시에 가능할 때
- 분석/리서치 여러 개를 동시에 돌릴 때
- 납품팀이 여러 클라이언트 콘텐츠를 뽑을 때
