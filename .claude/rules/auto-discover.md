# 자동 탐색 규칙 — 시스템 능력 활용

## 원칙
리안은 시스템에 뭐가 있는지 다 기억 못 한다. **내가 알아서 쓰거나 제안해야 한다.**

## 규칙 1: 태스크 받으면 CAPABILITIES 먼저 체크

리안이 "~해줘" 하면:
1. `.claude/CAPABILITIES.md` **먼저 읽는다**
2. 트리거 매핑에 있으면 → 해당 능력 즉시 사용
3. 없으면 → 유사 능력 탐색 (키워드 유사도)
4. 그래도 없으면 → "이런 기능 필요해 보여요. 설치할까요? 아니면 [X]로 대체할까요?" 제안

**예시:**
- 리안: "랜딩 만들어줘" → `CAPABILITIES.md` 디자인 섹션 → `Skill("landing")` 바로 실행
- 리안: "인스타 링크 분석해" → 인스타 섹션 → `analyze_instagram.py` 바로 실행
- 리안: "영상 편집 자동화해줘" → 매핑 없음 → "Premiere Pro 플러그인은 없어요. 간단한 영상이면 Kling MCP 쓰거나 fal.ai 붙이는 거 제안할까요?" 물어보기

## 규칙 2: 새 능력 추가 시 즉시 등록

설치/클론/생성 직후 **반드시**:
1. `.claude/CAPABILITIES.md` 트리거 매핑 + 전체 목록 업데이트
2. `.claude/INSTALLED.md` 동기화
3. 리안에게 "이제 '[트리거]' 하면 [X] 돼요" 한 줄 보고

**안 하면 묻힘.** 새 기능 설치하고 기록 안 하면 한 달 후 리안도 나도 모름.

## 규칙 3: 서브에이전트 스폰 시 CAPABILITIES 주입

에이전트(be/fe/cdo 등) 스폰할 때 프롬프트 시작 부분에:
```
먼저 `.claude/CAPABILITIES.md` 확인해서 쓸 수 있는 도구 파악.
```
한 줄 추가. 각 에이전트가 자기 작업에 필요한 스킬/툴 알아서 씀.

## 규칙 4: 주기적 감사 (drift 방지)

- `weekly_runner.py`가 월요일마다 `.claude/skills/`, `.claude/agents/`, `.mcp.json`, `company/tools/`, `company/utils/` 스캔
- CAPABILITIES.md에 없는 것 발견 → 보고사항들.md에 경고
- 리안이 모르는 사이에 생긴 것 즉시 감지

## 금지
- "스킬 있는 것 같은데..." 추측 금지 → 반드시 파일 확인
- CAPABILITIES.md 업데이트 누락한 채 설치 종료 금지
- 리안한테 "스킬 쓸까요?" 매번 물어보기 금지 → 명확하면 그냥 쓰기
