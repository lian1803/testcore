# 리안 요청 → 시스템 실행 트리거

## Git Push 트리거

| 리안이 말하면 | 실행 |
|---|---|
| "올려" / "push" / "깃에 올려" | `bash push.sh` (origin + core-shell 둘 다) |
| "개인깃 올려" | `bash push.sh origin` |
| "팀깃 올려" / "팀에 올려" | `bash push.sh team` (team/ 포함) |

**핵심 규칙:**
- `origin` (lian1803/testcore) = 개인깃 → team/ 제외
- `core-shell` (core-shell1/core-shell) = 팀깃 → team/ 포함
- `.gitignore`는 origin 기준. core-shell은 push.sh가 임시 브랜치로 team/ 강제 포함 처리



## 프로젝트 시작 트리거
리안이 아래 패턴으로 말하면 → 이사팀 소집:
- "~할거야" / "~프로젝트" / "~하고 싶어" / "~사업 해볼까" + 새 아이디어
- "이거 해줘" + 새로운 서비스/제품 설명

순서: "이사팀 소집할게. GO?" → `python company/main.py "아이디어"` → GO면 `build_team.py` → `run_{팀명}.py` → 보고사항들.md 안내

**트리거 제외** (기존 프로젝트): "어제에 이어서" / "계속" / "이거 고쳐줘" / 기존 파일 수정

## 기존 팀 실행 트리거

| 리안이 말하면 | 실행 |
|---|---|
| "영업팀 돌려줘" / "온라인영업" / "영업 스크립트" | `run_온라인영업팀.py "태스크"` |
| "납품팀 돌려줘" / "콘텐츠 뽑아줘" | `run_온라인납품팀.py "태스크"` |
| "마케팅팀 돌려줘" | `run_온라인마케팅팀.py "태스크"` |
| "오프라인 영업/마케팅" | `offline_sales.py "태스크"` |
| "일일 루프" / "daily" / "오늘 콘텐츠" | `daily_auto.py` |

모든 실행: `cd company && ./venv/Scripts/python.exe {스크립트}`
태스크 없으면 → "어떤 태스크로 돌릴까?" 한 줄만 물어봐.

## 디자인 트리거 (2가지 방법)

### 1번: R3F 3D 인터랙티브 히어로
리안이 아래 패턴으로 말하면 → `Skill("r3f-hero")` 실행:
- "3D 히어로" / "R3F" / "살아있는 3D" / "3D로 돌려줘"
- "날아다니는" / "움직이는 캐릭터" / "인터랙티브 3D"
- "1번으로 해줘" (디자인 맥락에서)

→ `.claude/skills/r3f-hero.md` 플로우 전체 실행.
→ Vite + React + R3F + GLTF 모델 + 착지 시퀀스.

### 2번: 영상 히어로 (작업 중 — 나중에 추가 예정)
리안이 아래 패턴으로 말하면:
- "영상 히어로" / "영상으로" / "2번으로 해줘" (디자인 맥락에서)

→ 추후 등록 예정.

### 일반 랜딩/사이트 (HTML)
리안이 아래 패턴으로 말하면 → `Skill("landing")` 실행:
- "디자인 해줘" / "사이트 만들어" / "랜딩 만들어" / "웹사이트 하나" / "홈페이지 만들어"
- "디자인 하나 해서 html로" / "html 하나 뽑아줘"

→ 반드시 `/landing` 체인 전체를 따라야 함.
→ director 스폰 필수 (스킵 금지).
