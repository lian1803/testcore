3번: 토지 개요·법률서를 넣으면 특례법·개발 가능성·투자금 범위를 뽑는 AI 분석 SaaS

보고서 핵심만 정리
2번 인테리어 비용절감 시스템
유형: 인테리어 시공업체용 B2B SaaS, 견적·자재내역·거래처·법률 리스크·공기 산출까지 한 번에.

시장: 국내 인테리어 30조, SAM 500억 추정, 월 10~30만 구독 모델.

페인포인트: 자재 미명시로 품질분쟁, 하자보수 조건 누락, 기타 항목 뭉뚱그려서 인건비 손실, 공기 예측 실패로 지체상금 등 레벨 4~5 문제를 돈 단위로 정리.

리스크: 7개 기능 동시 구현, 지역별 자재 단가 DB 없으면 정확도·법적 책임 이슈.

조건: MVP는 “자재 상세내역서 + 기본 견적서” 두 개만, “정확 견적”이 아니라 초안 자동화 포지셔닝, 조달청 표준품셈·공공데이터 활용 가능 여부 확인, API 비용·CAC 계산.

48시간 검증: 실제 견적 3~5개로 ±20% 오차 테스트 + 사장 5명 인터뷰로 월 10만 이상 지불 의향 체크.

3번 토지분석 시스템
유형: 시행사·건축사무소·법인 투자자용 B2B SaaS, 토지 개요·법률서 입력 → 특례법 해당, 개발 승인 확률, 투자 가능 금액, 정책 흐름.

시장: 연간 토지 거래 20만~30만 필지, 개발 검토 10만~20만건, 기존 컨설팅 100~500만/건 → 10만/건으로 치환 시 마진 80%+.

페인포인트: 특례법 수동 체크로 수억 손실, 대출 심사 지연, 승인확률 몰라 지방자치단체 계속 방문, 조례 변경 수동 추적.

리스크: AI 법률 분석 정확도 90%급 필요, 틀리면 손해 크고 법적 책임 이슈.

조건: 실제 문서 10건으로 AI vs 전문가 일치율 80% 이상이면 “참고용 AI + 전문가 최종 확인” 포지셔닝, 이용약관에 면책 명시, API 단가(500~2000원) vs CAC(5~20만) 넣어서 단위경제 확인.

48시간 검증: 토지 문서 5~10건 수집, AI 멀티에이전트 분석 vs 부동산 전문 변호사 비교 + 커뮤니티 설문(10만 이상 낼 의향).

종합 권고
둘 다 조건부 GO지만, 토지분석(3번)을 먼저 검증 추천.

이유: 점수 조금 더 높고, 마진 구조 명확, MVP 범위 좁고, 기존 컨설팅 200만 vs AI 10만으로 가격 메리트 큼.

관련 “법률·규제 대응용” 깃헙 레포 추천
너가 지금 하려는 건 “AI가 한국 법·조례·판례를 보고 리스크·특례·승인 확률을 판단해 주는 SaaS”라서, 크게 두 축이 필요해 보여:

한국 법률 텍스트·판례 데이터 및 벤치마크

국가법령정보 API를 AI 친화적으로 감싼 도구

1. 한국 법률 텍스트·판례·벤치마크
lbox-kr/lbox-open: 대규모 한국 판례 코퍼스(150k 케이스) + 사실관계→법령 매핑(statute classification) 등 여러 태스크 포함.

토지분석 쪽: “사실관계 → 관련 법령/조항 추천”, “판례 요약” 같은 기능에 바로 쓸 수 있는 구조.

lbox-kr/kbl (Korean Benchmark for Legal Language Understanding): 22만여 개 조문(법령 전문 덤프) + 판례 기반 법률 언더스탠딩 벤치마크.

법률 QA, 조문 검색 정확도, 법규 해석 관련 모델 튜닝·평가에 좋음.

onspark/LEAP_NLI_v2.0: 한국 법률 추론(NLI) 데이터셋, “조문/판례 vs 설명이 서로 모순인지·함의하는지·중립인지”를 보는 태스크.

“이 토지 개발 계획이 이 조항에 위반되냐/부합하냐” 같은 논리 체크 모델에 참고.

yeontaek/Korea-Law-LLM: 한국 법률 데이터 기반 LLM 프로젝트.

이미 법률 도메인으로 튜닝된 LLM 구조와 데이터 전처리 아이디어를 참고 가능.

LegalInsight/PretrainedModel: 한국어 법률 특화 BERT, 계약서·개인정보 NER 등 법률 도메인에 특화된 사전학습 모델과 샘플 데이터.

인테리어 견적 약관/계약서 리스크 태깅, 개인정보 관련 문장 NER 등에 재사용하기 좋음.

이 네트워크를 베이스로 “법률 리스크”·“특례법 해당 여부”를 판단하는 서브모델을 만들고, 위에 도메인 룰 + 규제 체크 로직만 얹으면 돼.

2. 국가법령·조례·판례 API 래퍼
chrisryugj/korean-law-mcp: 법제처 40여 개 Open API(법령·판례·조례·조약 등)를 14개 MCP 도구로 통합해 둔 레포.

특징:

search_law, get_law_text로 법령 전문 조회.

판례·헌재·조세심판 등 17개 도메인을 search_decisions(domain) / get_decision_text(domain) 두 개 체인으로 통합.

chain_full_research, chain_law_system, chain_document_review 같은 체인 도구로 “법령→판례→해석”까지 한 번에 묶는 구조.

토지분석 SaaS: 여기 API 체인을 그대로 백엔드에 붙이면

“특례법 후보 조문 검색 → 관련 판례 조회 → 요약”까지 한 번에 돌릴 수 있어서, 정확도·법적 근거 레퍼런스가 훨씬 탄탄해짐.

3. 참고용: 법률 계약서/리스크 분석 프로젝트
datawhales/Korean_Law: 한국어 법률 계약서에 필수 조항이 다 있나 체크하는 멀티라벨 분류 프로젝트.

인테리어 견적·계약서에 “하자보수 기간/책임”, “지체상금 규정” 등 필수 조항 누락 여부를 점검하는 모듈 만들 때 참고 가능.

일단 이 중에서 바로 포크해서 쓰고 싶은 건 korean-law-mcp(법제처 API 래퍼)랑 lbox-open or KBL(법률 데이터·벤치마크) 두 개일 것 같은데, 너는 지금 인테리어 쪽 먼저 손대고 싶어, 아니면 토지분석 쪽 먼저 세팅하는 게 좋아?

지금 실제로 있는 것들
대규모 한국 법률·판례 데이터셋

lbox-kr/lbox-open: 형사·민사 사건 판례 15만 건 코퍼스 공개, LBox가 AGI Korea 같은 데서도 홍보했던 그 데이터셋.

lbox-kr/kbl: EMNLP Findings 2024에 나온 KBL 벤치마크 공식 레포, 한국 법령 22만 조문 + 판례 기반으로 7개 지식 태스크, 4개 추론 태스크, 변호사 검증까지 들어간 벤치마크.

국가법령정보센터 Open API를 감싼 MCP 서버

korean-law-mcp: 법제처 39개 API를 14~80여 개 도구로 묶어서, 법령·판례·자치법규·조약·행정규칙까지 AI 에이전트에서 바로 쓰게 해놓은 MCP 툴.

설치해서 korean-law "민법 제1조" 이런 식으로 바로 쿼리 날리거나, Claude·Cursor 같은 에이전트에서 한국 법령 실시간 조회용으로 연동하는 예시까지 들어있음.

법률 특화 모델·계약서 분석 쪽

LBox 쪽 KBL는 “한국 법률 언어 이해 평가” 벤치마크라, 네가 토지분석 AI 만들 때 “법률 이해 성능”을 객관적으로 재볼 수 있는 셈.

별도의 레포로 한국 법률 계약서 조항 체크, 법률 특화 BERT 모델들도 이미 공개된 것들이 있고, 그 중 하나로 LegalInsight/PretrainedModel 같은 한국어 법률 특화 모델 레포가 있음.

그래서 네 아이디어에서 “법령·판례·조례를 자동으로 찾아보고 해석하는 모듈” 자체는 완전 바닥부터 만들 필요 없고,

데이터/벤치마크는 LBox 쪽(kbl, lbox-open)

실시간 법령/판례 조회는 korean-law-mcp
를 꽂아서, 그 위에 인테리어/토지 도메인 룰이랑 UX만 얹는 구조로 가는 게 가능해.

원하면, korean-law-mcp 기준으로
“토지 특례법 체크 체인 하나, 인테리어 공사 표준약관 체크 체인 하나”
이렇게 딱 2개 워크플로우 설계해서, 아예 네 SaaS 백엔드 초안 구조를 코드 레벨로 그려줄까?
