## 1. 컬러 팔레트

```
primary:         #1E3A5F   (딥블루 — 브랜드 메인, 신뢰·안정감)
primary-light:   #2D5F8A   (딥블루 밝은 변형 — 호버, 보조 강조)
accent:          #22C55E   (절세 그린 — 절세 포인트, 성공 상태)
accent-hover:    #16A34A   (그린 호버)
background:      #F8FAFC   (전체 배경 — 차갑지 않은 흰색)
surface:         #FFFFFF   (카드, 모달, 사이드바 배경)
text-primary:    #1A202C   (본문 주요 텍스트)
text-secondary:  #718096   (보조 텍스트, 플레이스홀더)
border:          #E2E8F0   (구분선, 인풋 테두리)
error:           #EF4444   (에러, 경고 강조)
warning:         #F59E0B   (주의 상태, 법적 고지 배너)
success:         #22C55E   (성공 상태 — accent와 동일)
```

### 컬러 사용 원칙
- `primary`는 버튼, 링크, 네비게이션 활성 상태에 사용
- `accent`는 절세 관련 수치, 절감액 강조, CTA 보조에 사용
- `warning`은 법적 고지 배너, 한도 초과 알림에 사용
- `error`는 폼 오류, OCR 실패 상태에만 사용
- 배경은 항상 `background(#F8FAFC)` 위에 `surface(#FFFFFF)` 카드 구조로 레이어링

## 2. 타이포그래피

**기본 폰트**: Pretendard (한국어 최적화, variable font 권장)
**폴백**: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

```css
/* 폰트 임포트 */
@import url('https://cdn.jsdelivr.net/gh/orioncactus/pretendard/dist/web/variable/pretendardvariable.css');

font-family: 'Pretendard Variable', Pretendard, sans-serif;
```

### 사이즈 체계 (Tailwind 기준)

| 역할 | 클래스 | 크기 | 굵기 | 용도 |
|------|--------|------|------|------|
| Display | `text-4xl` | 36px | 700 (Bold) | 랜딩 Hero 헤드라인 |
| H1 | `text-3xl` | 30px | 700 (Bold) | 페이지 제목 |
| H2 | `text-2xl` | 24px | 600 (SemiBold) | 섹션 제목 |
| H3 | `text-xl` | 20px | 600 (SemiBold) | 카드 제목, 서브섹션 |
| H4 | `text-lg` | 18px | 500 (Medium) | 강조 본문 |
| Body | `text-base` | 16px | 400 (Regular) | 일반 본문 |
| Small | `text-sm` | 14px | 400 (Regular) | 보조 정보, 레이블 |
| XSmall | `text-xs` | 12px | 400 (Regular) | 뱃지, 캡션, 타임스탬프 |

### 텍스트 컬러 원칙
- 헤드라인: `text-primary(#1A202C)`
- 본문: `text-primary(#1A202C)`
- 보조 설명: `text-secondary(#718096)`
- 링크: `primary(#1E3A5F)`, 호버 시 `primary-light(#2D5F8A)`
- 강조 수치 (절세액 등): `accent(#22C55E)`

## 3. 화면별 설계 명세

### 3-1. 랜딩 페이지 (`/`)

**목적**: 첫 인상 + 가치 전달 + 무료 회원가입 전환

#### 레이아웃 구조
```
[네비게이션 바]
[Hero 섹션]
[기능 소개 카드 3개]
[신뢰 지표 섹션]
[가격 플랜 섹션]
[이용 후기 섹션]
[푸터]
```

#### 네비게이션 바
- 좌측: 서비스 로고 (텍스트 + 아이콘)
- 우측: "로그인" (ghost 버튼) + "무료로 시작하기" (primary 버튼)
- 스크롤 시 `shadow-sm` + `backdrop-blur` 적용 (sticky)
- 높이: `h-16` (64px)

#### Hero 섹션
- 배경: `background(#F8FAFC)` 또는 `primary(#1E3A5F)` 다크 배경 + 흰 텍스트
- **헤드라인**: "영수증 찍으면 세금신고 끝" (Display, Bold)
- **서브카피**: "AI가 경비를 자동 분류하고 종합소득세 신고서 초안을 만들어드려요. 1인 사업자·프리랜서를 위한 절세 파트너." (text-lg, text-secondary)
- **CTA 버튼**: "무료로 시작하기" — accent(#22C55E) 배경, 흰 텍스트, `rounded-xl`, `px-8 py-4`, 호버 시 `accent-hover`
- **보조 문구**: "신용카드 불필요 · 영수증 20건 무료" (text-sm, text-secondary)
- **Hero 이미지/목업**: 우측에 앱 스크린샷 또는 영수증→신고서 변환 일러스트
- 최소 높이: `min-h-[600px]`, 세로 중앙 정렬

#### 기능 소개 카드 (3개)
- 배열: `grid grid-cols-3 gap-6` (모바일: `grid-cols-1`)
- 각 카드: `surface(#FFFFFF)` 배경, `rounded-2xl`, `p-8`, `shadow-sm`, border: `#E2E8F0`
- 아이콘: primary 계열 색상, 48×48px

| 카드 | 아이콘 | 제목 | 설명 |
|------|--------|------|------|
| 1 | 카메라/스캔 | OCR 자동 인식 | 영수증 사진 한 장으로 날짜·금액·상호명 자동 추출, 정확도 97% |
| 2 | 태그/분류 | AI 자동 분류 | GPT-4o가 업종에 맞게 경비 항목 자동 분류. 오분류 시 한 번에 수정 |
| 3 | 문서/다운로드 | 신고서 초안 생성 | 종합소득세 신고서 PDF/Excel 자동 생성. 홈택스 직접 입력 가이드 포함 |

#### 신뢰 지표 섹션
- 배경: `primary(#1E3A5F)`, 흰 텍스트
- 3개 수치 가로 나열: `grid grid-cols-3`
  - **OCR 정확도 97%** / **영수증 처리 10초** / **누적 절세 금액 X억원**
- 수치: `text-4xl font-bold`, 설명: `text-sm text-white/70`

#### 가격 플랜 섹션
- 제목: "간단하고 투명한 요금제"
- 카드 2개 나란히: 무료 플랜 / 프리미엄 플랜
- 추천 플랜에 `ring-2 ring-accent` 강조 + "인기" 뱃지 (accent 배경)

| | 무료 | 프리미엄 |
|--|------|---------|
| 가격 | 0원 | 월 9,900원 / 연 79,200원 |
| 영수증 | 월 20건 | 무제한 |
| 신고서 | 연 1회 | 무제한 |
| CTA | "무료로 시작" | "지금 구독하기" |

### 3-2. 회원가입 (`/signup`) / 로그인 (`/login`)

**목적**: 빠르고 안전한 계정 생성, 진입 장벽 최소화

#### 레이아웃
- 중앙 카드 레이아웃: `max-w-md mx-auto`, 세로 중앙 `min-h-screen flex items-center`
- 좌측 브랜드 영역(데스크탑): 절반은 `primary(#1E3A5F)` 배경 + 가치 메시지
- 카드: `surface(#FFFFFF)`, `rounded-2xl`, `shadow-lg`, `p-8`

#### 주요 컴포넌트
- **로고**: 상단 중앙
- **제목**: "무료로 시작하세요" / "다시 오셨네요"
- **소셜 로그인 버튼** (상단 배치 권장):
  - 카카오: `bg-[#FEE500]` + 카카오 아이콘, `rounded-xl`, `w-full`
  - 구글: `border border-border bg-white` + 구글 아이콘, `rounded-xl`, `w-full`
- **구분선**: "또는 이메일로 계속하기" (`text-secondary` 가운데 텍스트)
- **이메일 Input**: label "이메일 주소", placeholder "you@example.com"
- **비밀번호 Input**: label "비밀번호", 8자 이상 + 특수문자, 눈 아이콘 토글
- **이용약관 체크박스** (회원가입만): "이용약관 및 개인정보처리방침에 동의합니다" — 링크 포함
- **제출 버튼**: "회원가입하기" / "로그인" — `w-full`, `primary` 배경
- **하단 링크**: "이미 계정이 있으신가요? 로그인" / "비밀번호를 잊으셨나요?"

#### 인터랙션
- 폼 유효성 검사: 실시간 (포커스 아웃 시)
- 에러: input 하단 `text-error text-sm` + input `border-error`
- 제출 중: 버튼 로딩 스피너 + 비활성화
- 성공: `/onboarding` (신규) 또는 `/dashboard` (기존) 리다이렉트

### 3-3. 온보딩 (`/onboarding`)

**목적**: 사용자 세금 프로필 수집 → 정확한 공제 계산 기반 마련

#### 레이아웃
- 중앙 카드, `max-w-lg mx-auto`
- **스텝 인디케이터** 상단 고정: 1/3 → 2/3 → 3/3
  - `Progress` 컴포넌트 (`h-2`, `bg-primary`) + 스텝 번호 텍스트
  - 완료 스텝: `primary`, 현재 스텝: `primary-light`, 미완료: `border`

#### 스텝 구성

**스텝 1 — 업종 선택**
- 제목: "어떤 일을 하세요?"
- `Select` 컴포넌트: 국세청 업종코드 기반 드롭다운
  - 예: 프리랜서 (소프트웨어 개발), 유튜버/크리에이터, 디자이너, 강사, 배달라이더, 소매업, 서비스업 등
- 선택 시 관련 설명 텍스트 표시 (어떤 경비가 공제되는지 미리 안내)

**스텝 2 — 과세 유형 및 사업자 정보**
- 제목: "사업자 정보를 입력해주세요"
- 과세유형 라디오 그룹:
  - 일반과세자 / 간이과세자 / 면세사업자 / 개인(사업자등록 없음)
  - 각 옵션 하단 설명 텍스트 (text-secondary, text-sm)
- 사업자등록번호 Input (선택, 미입력 시 프리랜서 처리)
  - placeholder: "000-00-00000 (선택사항)"

**스텝 3 — 신고 연도 선택**
- 제목: "몇 년도 세금을 준비하시나요?"
- `Select`: 신고 대상 연도 (기본값: 직전 연도)
- 하단 안내 박스: "종합소득세 신고 기간: 매년 5월 1일 ~ 5월 31일" (`warning` 배경 옅게)
- "시작하기" 버튼 → `/dashboard` 이동

#### 인터랙션
- 이전/다음 버튼 쌍
- 스텝 전환: 슬라이드 애니메이션 (`translate-x` 트랜지션)
- 다음 버튼: 필수 항목 미선택 시 비활성화

### 3-4. 대시보드 (`/dashboard`)

**목적**: 경비 현황 한눈에 파악 + 핵심 액션(영수증 추가, 신고서 생성) 유도

#### 레이아웃 — 사이드바 + 콘텐츠
```
[사이드바 — 고정, w-64]  |  [메인 콘텐츠 — flex-1]
                          |  [상단 헤더 바]
                          |  [KPI 카드 영역]
                          |  [차트 + 최근 영수증 리스트]
```

#### 사이드바
- 배경: `surface(#FFFFFF)`, 우측 `border-r border-border`
- 상단: 로고 + 서비스명
- 네비게이션 메뉴 (아이콘 + 텍스트):
  - 대시보드 (홈 아이콘) — 활성 시 `primary` 배경 + 흰 텍스트, `rounded-lg`
  - 영수증 (카메라 아이콘)
  - 경비 목록 (목록 아이콘)
  - 신고서 (문서 아이콘)
  - 설정 (기어 아이콘) — 하단 고정
- 하단: 사용자 프로필 (아바타 + 이름 + 플랜 표시)
- 무료 플랜 한도 표시: Progress 바 "영수증 12/20건 사용"

#### 상단 헤더 바
- 페이지 타이틀: "대시보드"
- 우측: "영수증 추가" 버튼 (primary) + 알림 아이콘

#### KPI 카드 (4개, grid-cols-4)
각 카드: `surface` 배경, `rounded-xl`, `p-6`, `shadow-sm`

| 카드 | 수치 색상 | 아이콘 |
|------|----------|--------|
| 올해 총 경비 | `text-primary` | 영수증 |
| 예상 세액 | `text-error` | 계산기 |
| 등록 영수증 수 | `text-primary` | 카메라 |
| 절세 예상액 | `text-accent` (그린) | 절약 |

- 카드 하단: 전월 대비 변화 (▲/▼ + %, `text-sm`)
- "절세 예상액" 카드는 `border-l-4 border-accent` 강조

#### 월별 경비 바 차트
- `Recharts` `BarChart`
- X축: 월 (1월~12월), Y축: 금액 (만원 단위)
- 바 색상: `primary(#1E3A5F)`, 호버 시 `primary-light`
- 카테고리별 누적 바(Stacked) 선택 가능 (토글)
- 카드 제목: "월별 경비 현황" + 연도 선택 드롭다운

#### 최근 영수증 리스트
- 우측 하단 또는 차트 아래 배치
- 최근 5건 표시, "전체 보기" 링크
- 각 항목: [썸네일] [상호명 + 날짜] [카테고리 뱃지] [금액]
- 금액: `text-right font-semibold`

#### FAB (Floating Action Button)
- 모바일 전용: 우측 하단 고정, `primary` 배경, + 아이콘
- 클릭 → `/receipts/new`

### 3-5. 영수증 업로드 + OCR 결과 (`/receipts/new`)

**목적**: 영수증 1장을 빠르게 등록 (목표 10초 이내)

#### 레이아웃
- 단일 컬럼, `max-w-2xl mx-auto`
- 상단: 뒤로가기 + 페이지 제목 "영수증 추가"

#### 업로드 영역
- 드래그앤드롭 존:
  - 점선 테두리 `border-2 border-dashed border-border`, `rounded-2xl`, `p-12`
  - 중앙: 업로드 아이콘 + "여기에 영수증을 드래그하거나" + "사진 선택" 버튼 (outline)
  - 드래그 오버 시: `border-primary bg-primary/5`
- 카메라 촬영 버튼: "카메라로 촬영" (모바일에서 주력)
- 지원 형식 안내: "JPG, PNG, HEIC · 최대 10MB" (text-xs, text-secondary)

#### OCR 처리 로딩 상태
- 업로드 후 전체 카드 위에 오버레이
- 로딩 애니메이션: 스캔 라인이 내려오는 애니메이션 (영수증 이미지 위)
- 텍스트: "AI가 영수증을 읽고 있어요... (최대 10초)" (progress 바 포함)
- 단계 표시: "이미지 분석 중" → "텍스트 인식 중" → "카테고리 분류 중"

#### OCR 결과 카드
- 좌측: 업로드된 영수증 이미지 썸네일 (`rounded-lg`, 클릭 시 확대)
- 우측: 추출된 정보 (수정 가능)

| 필드 | 컴포넌트 | 비고 |
|------|----------|------|
| 날짜 | `Input` type="date" | OCR 결과 자동 채움 |
| 금액 | `Input` type="number" | 원 단위, 숫자만 |
| 상호명 | `Input` type="text" | OCR 결과 자동 채움 |
| 카테고리 | `Select` 드롭다운 | AI 분류 결과 + 수동 변경 가능 |
| 사업 관련 여부 | `Switch` 토글 | 기본값: ON |
| 메모 | `Textarea` (선택) | 추가 설명 |

- OCR 신뢰도 낮은 필드: 노란 배경 + "확인 필요" 뱃지
- AI 분류 근거: "식비(업무용)으로 분류됨 — 상호명 '스타벅스' 기반" (text-sm, text-secondary)
- 저장 버튼: "저장하기" (primary, w-full) + "취소" (ghost)

### 3-6. 경비 목록 (`/expenses`)

**목적**: 등록된 모든 경비 확인 + 수정 + 필터링

#### 레이아웃
- 사이드바 + 메인 콘텐츠 (대시보드와 동일 Shell)
- 상단: 페이지 제목 + 필터 영역 + "영수증 추가" 버튼

#### 필터 영역
- 월별 필터: `Select` (전체 / 1월 / 2월 ... / 12월)
- 카테고리 필터: `Select` (전체 / 식비 / 교통비 / 통신비 / 사무용품 / 교육비 / 기타)
- 검색: `Input` (상호명, 메모 검색)
- 정렬: 날짜 내림차순 기본

#### 카테고리 뱃지 색상 체계

| 카테고리 | 배경색 | 텍스트 |
|----------|--------|--------|
| 식비 | `#FEF3C7` | `#92400E` |
| 교통비 | `#DBEAFE` | `#1E40AF` |
| 통신비 | `#F3E8FF` | `#6B21A8` |
| 사무용품 | `#DCFCE7` | `#166534` |
| 교육비 | `#FFE4E6` | `#9F1239` |
| 기타 | `#F1F5F9` | `#475569` |

#### 테이블
- 컬럼: [날짜] [상호명] [카테고리] [금액] [사업관련] [액션]
- 행 높이: `h-14`, 짝수 행 `bg-gray-50/50`
- 인라인 수정: 행 클릭 시 편집 모드 (`Dialog` 또는 인라인 토글)
- 금액: 우측 정렬, `font-semibold`
- 액션: 수정 아이콘(펜) + 삭제 아이콘(휴지통) — 행 호버 시 표시
- 삭제 시: 확인 `Dialog` (실수 방지)
- 하단: 페이지네이션 (20건/페이지 기본)

#### 합계 표시
- 테이블 상단 또는 하단: 필터된 기간의 총 경비 합계 (text-lg font-bold)

### 3-7. 신고서 생성 (`/tax-return`)

**목적**: 종합소득세 신고서 초안 단계별 생성 + 다운로드

#### 레이아웃
- Wizard UI: 단계별 진행, 전체 폭 `max-w-3xl mx-auto`
- 상단: 스텝 인디케이터 (4단계)

#### 스텝 구성

**스텝 1 — 소득 정보 입력**
- 제목: "올해 수입을 입력해주세요"
- 연간 총 매출액 Input (필수)
- 기타 소득 Input (선택 — 이자소득, 배당소득 등)
- 부양가족 수 Input (인적공제 자동 계산용)
- 안내 박스: "정확한 수입 금액은 통장 입금 내역 또는 계산서를 참고해주세요" (info 스타일)

**스텝 2 — 경비 확인**
- 제목: "등록된 경비를 확인해주세요"
- 카테고리별 경비 합계 표 (읽기 전용, 수정 불가 — 경비 목록에서 수정하도록 유도)
- 총 경비 합계: `text-2xl font-bold text-accent`
- "경비 목록에서 수정" 링크
- 경비 0건인 경우: "아직 등록된 영수증이 없어요" + "영수증 추가하기" 버튼

**스텝 3 — 신고서 미리보기**
- 제목: "신고서 초안을 확인하세요"
- 카드 형태로 주요 항목 표시:
  - 총 수입 / 총 경비 / 소득금액 / 각종 공제 / 과세표준 / 산출세액 / **예상 납부세액**
- 예상 납부세액: `text-3xl font-bold text-error` (크게 강조)
- 절세 예상액 (경비 미등록 대비): `text-accent` 강조 + 절세 팁 안내
- **법적 고지 배너** (필수):
  ```
  [!] 이 신고서는 참고용 초안입니다.
  본 서비스는 세금 신고 준비를 돕는 도구로, 세무 대리 서비스가 아닙니다.
  최종 신고 및 납부의 법적 책임은 사용자에게 있습니다.
  ```
  - 스타일: `bg-warning/10 border border-warning rounded-xl p-4`, 아이콘 ⚠️
  - 위치: 미리보기 카드 상단 고정

**스텝 4 — 다운로드**
- 제목: "신고서를 다운로드하세요"
- 다운로드 버튼 2개:
  - "PDF로 다운로드" (primary 버튼)
  - "Excel로 다운로드" (outline 버튼)
- **홈택스 입력 가이드** 섹션:
  - "국세청 홈택스에서 직접 입력하는 방법"
  - 단계별 스크린샷 + 번호 리스트
  - Accordion 컴포넌트로 접기/펼치기
- 완료 후: "대시보드로 돌아가기" 버튼

## 5. 반응형 브레이크포인트

Tailwind CSS 기본 브레이크포인트 기준:

| 브레이크포인트 | 최소 너비 | 적용 대상 |
|--------------|----------|----------|
| `sm` | 640px | 모바일 가로, 소형 태블릿 |
| `md` | 768px | 태블릿 |
| `lg` | 1024px | 소형 데스크탑, 사이드바 등장 |
| `xl` | 1280px | 일반 데스크탑, KPI 카드 4열 |
| `2xl` | 1536px | 와이드 모니터 (max-w 제한으로 레이아웃 고정) |

### 화면별 반응형 처리

#### 랜딩 페이지
```
기본(모바일): 단일 컬럼, Hero 텍스트 중앙 정렬, 기능 카드 1열
md: Hero 이미지 등장, 기능 카드 2열
lg: 기능 카드 3열, 신뢰 지표 3열
```

#### 대시보드
```
기본~md: 사이드바 숨김, 하단 탭바 네비게이션으로 대체, KPI 카드 2열
lg: 사이드바 고정 표시 (w-64), KPI 카드 4열
xl: 차트와 최근 영수증 2열 병렬 배치
```

#### 경비 목록 테이블
```
기본~md: 카드 리스트 뷰 (테이블 대신)
lg: 테이블 뷰 전환
```

#### 영수증 업로드/OCR
```
기본: 업로드 영역 + OCR 결과 세로 스택
md: 좌측 이미지, 우측 결과 2열 배치
```

#### 신고서 Wizard
```
기본: 스텝 인디케이터 점 형태 (텍스트 숨김)
md: 스텝 텍스트 표시, 폼 필드 2열 배치 가능
```

### 공통 패턴
- 컨테이너: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
- 카드 그리드: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-4 lg:gap-6`
- 텍스트 크기: 모바일에서 1단계 작게 (예: `text-3xl md:text-4xl`)
- 버튼: 모바일에서 `w-full`, 데스크탑에서 `w-auto`

## 6. 공통 UX 원칙

### 법적 포지셔닝 (전체 적용 필수)
- 모든 신고 관련 화면에 "신고 준비 도구" 문구 포함
- 신고서 생성 전/후: 법적 고지 배너 (warning 스타일) 필수 표시
- "세무 대리 행위 아님" 명시, "최종 책임은 사용자에게" 문구 포함

### 마이크로 인터랙션
- 버튼 호버: `transition-colors duration-200`
- 카드 호버: `shadow-md transition-shadow duration-200`
- 페이지 전환: `fade-in` 애니메이션 (150ms)
- OCR 로딩: 스캔 라인 애니메이션 (신뢰감 제공)

### 에러 처리
- OCR 실패: "인식에 실패했어요. 더 밝은 환경에서 다시 시도해주세요" + 재시도 버튼
- 네트워크 에러: `Toast` — "연결이 불안정해요. 잠시 후 다시 시도해주세요"
- 한도 초과: 페이월 `Dialog` — 부드럽게 업그레이드 유도 (강요 느낌 최소화)

### 빈 상태 (Empty State)
- 대시보드 (영수증 0건): 일러스트 + "첫 영수증을 추가해보세요" + CTA 버튼
- 경비 목록 (필터 결과 없음): "해당 조건의 경비가 없어요" + 필터 초기화 버튼

### 접근성
- 모든 인터랙티브 요소: 키보드 접근 가능 (`focus-visible` 스타일 유지)
- 색상만으로 정보 전달 금지 (아이콘 또는 텍스트 병행)
- 폼 에러: `aria-invalid`, `aria-describedby` 속성 사용
- 이미지: `alt` 텍스트 필수
