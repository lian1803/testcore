## 🔧 다음 단계 에이전트 지시사항

### **Frontend Developer에게**:
1. `LiquidChrome.jsx` 파일 열어서 OGL 셰이더 추출
2. Vanilla WebGL로 포팅 (GLSL 그대로 사용, Three.js 의존성 제거)
3. 5개 섹션 HTML 구조 작성 (`<section id="hero">` ~ `<section id="cta_final">`)
4. CountUp 숫자 애니메이션 구현 (Intersection Observer 사용)

### **Shader Engineer에게**:
1. LiquidChrome 셰이더의 `uniforms` 파라미터 확인
2. 마우스 인터랙션 코드 분리 (`u_mouse`, `u_time`)
3. 스크롤 진행도에 따른 `u_intensity` 변화 로직 추가

### **Motion Designer에게**:
1. GSAP ScrollTrigger로 5단계 시네마틱 구현
2. 각 섹션 전환 시 `perspective` 회전 적용
3. 카드 cascade 애니메이션: stagger 200ms, y: 100 → 0
