---

## 개요

온보딩 프로세스는 계약 체결 후 클라이언트가 우리의 데이터 분석 서비스를 원활하게 시작할 수 있도록 30일간 구조화된 프로세스입니다.

**목표**: 
- Week 1: GA4 + 픽셀 완전 설치 및 데이터 수집 확인
- Week 2: 커스텀 이벤트 설정 완료 및 데이터 검증
- Week 3: 대시보드 개발 완료 및 팀 교육
- Week 4: 첫 번째 리포트 제공 및 운영 시작

## Day 8-14: 커스텀 이벤트 설계 (Week 2)

### Day 8-9: 이벤트 정의서 작성

**목표**: 비즈니스 목표에 맞춘 커스텀 이벤트 정의

**LIAN 할 일:**
- [ ] 클라이언트와 이벤트 정의 회의 (1시간)

  - 추적할 주요 액션 목록화
- [ ] 이벤트 정의서(Tracking Plan) 초안 작성
  
**이벤트 정의 예시 (이커머스):**

| 이벤트명 | 설명 | 발생 조건 | 파라미터 | 우선순위 |
|---------|------|---------|---------|---------|
| product_view | 상품 상세 페이지 방문 | 상품 페이지 로드 | product_id, product_name, category | P1 |
| add_to_cart | 장바구니 담기 | "장바구니 담기" 버튼 클릭 | product_id, product_name, price | P1 |
| purchase | 구매 완료 | 결제 성공 페이지 로드 | transaction_id, value, currency, items | P1 |
| lead | 상담 신청 | "상담 신청" 폼 제출 | lead_type, phone_number | P1 |
| scroll | 페이지 50% 이상 스크롤 | 페이지 절반 이상 스크롤 | page_path, scroll_percentage | P2 |
| video_play | 비디오 재생 | 상품 설명 영상 재생 | video_id, video_title | P2 |
| button_click | 특정 버튼 클릭 | CTA 버튼 클릭 | button_id, button_label | P3 |

- [ ] 파라미터(parameters) 정의:
  - 각 이벤트별 필요한 데이터 정의
  - 데이터 형식 및 범위 명시 (텍스트/숫자/날짜)
  - 예시 데이터 제공

**클라이언트 할 일:**
- [ ] 이벤트 정의서 검토 및 피드백 (48시간 내)
- [ ] 추가/삭제할 이벤트 제시
- [ ] 최종 승인

**점검 항목:**
- [ ] 이벤트 우선순위 합의
- [ ] 파라미터 명확성 확인

**산출물:**

### Day 15-16: 대시보드 요구사항 수집

- [ ] 대시보드 요구사항 회의 (1시간)
  - 대상 사용자별 필요한 정보 파악

**대시보드 1: Executive Summary (CEO/CMO 대상)**
- 주요 지표: 월 ROAS, CPA, 매출, 광고비 vs 매출 비교
- 채널별 성과 비율 (메타 50%, 구글 30%, 기타 20% 등)
- 추이 그래프: 월별 ROAS 추이, 월별 CPA 추이

- 채널별 상세 분석: 메타, 구글, 네이버, 카카오 각각
- 각 채널: 지출, 클릭, 전환, CPA, ROAS, CTR
- 기간별 비교 (어제 vs 지난주 vs 지난달)

**대시보드 3: 일일 모니터링 (광고 담당자 대상)**
- 어제/오늘 실시간 데이터: 지출, 클릭, CPC, 전환, CPA
- 시간대별 성과 그래프

**클라이언트 할 일:**
