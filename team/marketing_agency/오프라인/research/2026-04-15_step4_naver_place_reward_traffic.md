## 1. 퀴즈/정답 미션 리워드 — 소소분류

> 퀴즈 미션 = 사용자가 네이버 플레이스 내부에서 실제 정보를 확인해야 리워드가 지급되는 구조.
> 단순 클릭이 아닌 "정보 탐색 행동"을 유도한다는 점에서 2026 알고리즘과 표면적으로 궁합이 맞다.

### 1-a. 영업시간 확인 미션
- 한 줄 정의: 사용자가 플레이스 내 영업시간 탭을 확인하는 행동을 미션으로 설계, 완료 시 리워드 지급
- 미션 유형: 탐색형 (정보 확인) — 클릭 이상의 탐색 필요. 탭 진입 후 정보 노출 화면 도달이 조건.
- 출처 URL: https://www.i-boss.co.kr/ab-2987-494474 (퍼플페퍼 길찾기 미션 확장 게시물에서 영업시간 확인을 4대 미션 중 하나로 명시)
- 합법성: [x] 회색 — 실제 사용자가 수행하나, 리워드로 유도된 행동이므로 자연 신호 왜곡
- SKU 여부: [x] SKU — "영업시간 미션 리워드"로 독립 판매 단가 책정 가능. 타 미션과 단가 다르게 매겨짐.

### 1-b. 메뉴 탐색 미션
- 한 줄 정의: 플레이스 내 메뉴 탭 또는 메뉴 사진 섹션을 탐색하는 행동을 미션으로 설계
- 미션 유형: 탐색형 (메뉴 찾기) — 퍼플페퍼 4대 미션 중 "메뉴 찾기" 항목으로 실제 운영 중
- 출처 URL: https://www.i-boss.co.kr/ab-2987-494474 (퍼플페퍼), https://www.i-boss.co.kr/ab-2987-516347 (퍼플페퍼 신규리워드)
- 합법성: [x] 회색 — 메뉴 정보 확인이라는 자연 행동이나 리워드 유도로 인위성 존재
- SKU 여부: [x] SKU — 식음료 업종 전용 옵션으로 분리 판매 가능. 업종 특화 단가 책정 구조 확인됨.

### 1-c. 가격/정보 확인 미션
- 한 줄 정의: 플레이스 내 가격 정보, 상세 설명 등을 확인하는 행동을 미션으로 설계
- 미션 유형: 탐색형 (가격·정보 확인) — 퍼플페퍼 "가격·정보 확인" 미션으로 명시
- 출처 URL: https://www.i-boss.co.kr/ab-2987-516347 (퍼플페퍼 신규리워드 구조 설명)
- 합법성: [x] 회색 — 구조상 자연스러우나 리워드 지급 주체가 존재
- SKU 여부: [x] SKU — 전문직/의료 등 가격 민감 업종 대상 별도 상품 구성 가능. 단가 차이 발생.

### 1-d. 길찾기 실행 미션
- 한 줄 정의: 플레이스 내 길찾기 버튼을 실행(도보/자전거/대중교통 선택 포함)하는 행동을 미션으로 설계
- 미션 유형: 행동형 (길찾기 실행) — 퍼플페퍼 4대 미션 중 하나로 명시. 도보·자전거 선택 조건 포함
- 출처 URL: https://www.i-boss.co.kr/ab-2987-494474 (퍼플페퍼 길찾기 미션 확장 신규 게시물 제목)
- 합법성: [x] 어뷰징 경계 — 길찾기 실행은 방문 의사 신호인데, 리워드로 대량 생성 시 알고리즘 직접 기만에 해당
- SKU 여부: [x] SKU — "길찾기 리워드"로 3차 소분류에서도 이미 독립 SKU 확인. 퍼플페퍼 게시물 제목이 이를 상품화한 증거.

### 1-e. 복합 탐색 미션 (3단계 이상 순차 수행)
- 한 줄 정의: 영업시간 확인 → 메뉴 탐색 → 길찾기 실행 등 3단계 이상 행동을 순서대로 수행해야 리워드 지급
- 미션 유형: 복합형 (순차 탐색+행동) — 트리플 미션 구조. "검색 → 메뉴 3가지 이상 클릭 → 지도 탭 클릭 후 종료"가 실제 운영 사례
- 출처 URL: https://www.i-boss.co.kr/ab-2987-477812 (트리플 미션 리워드 구조 명시 — "메뉴 3가지 이상 클릭"이 중간 단계)
- 합법성: [x] 어뷰징 경계 — 복수 행동의 연계 설계 자체가 알고리즘 체류시간 조작과 직결
- SKU 여부: [x] SKU — "트리플 미션 리워드"라는 명칭으로 업계 내 독립 상품화 확인. 단가가 단일 미션보다 높게 책정됨.

### 1-f. 자동 검증 미션 리워드 (앱 기반 자동 완료 처리)
- 한 줄 정의: 토스/OK캐시백/캐시슬라이드/망고 등 리워드 앱이 미션 완료를 자동으로 검증하고 리워드를 지급하는 구조
- 미션 유형: 자동 검증형 — 리워드 앱이 클릭/탐색 완료를 자동으로 트래킹
- 출처 URL: https://www.marketing-center.co.kr/wp/%EB%84%A4%EC%9D%B4%EB%B2%84%ED%94%8C%EB%A0%88%EC%9D%B4%EC%8A%A4-%EB%A6%AC%EC%9B%8C%EB%93%9C-%ED%8A%B8%EB%A0%88%ED%94%BD%EC%9D%B4%EB%9E%80/ (토스/OK캐시백/캐시슬라이드/망고 매체 언급)

- SKU 여부: [x] SKU — 매체(토스 vs 캐시슬라이드 등)별로 단가가 다르게 책정됨. 매체 선택이 독립 상품 변수.

- 합법성: [x] 회색 — 인간이 직접 수행하므로 봇 리스크는 낮으나 리워드 유도라는 본질은 동일

## 2. 복합 행동 번들 리워드 — 소소분류

> 번들 = 단일 행동 리워드를 2개 이상 묶어 하나의 상품으로 판매하는 구조.

- 번들 구성: 유입(검색+클릭) + 체류(페이지 시간 확보)
- 실제 상품명: "네이버 플레이스 리워드 트래픽 구매 (유입 + 체류)" — 아반마케팅에서 해당 명칭으로 직접 판매 확인

- 합법성: [x] 회색 — 체류시간 신호를 리워드로 생성. 알고리즘 기만이나 봇은 아님.
- SKU 여부: [x] SKU — "유입+체류" 단독 상품으로 아반마케팅에서 독립 판매 확인.

- 출처 URL: https://j-stagram.com/blog/naver-place-reward-traffic-save (맛집/카페 전용으로 업종 분화까지 확인)
- 합법성: [x] 회색 — 저장하기는 강한 관심 신호인데 리워드로 생성 시 신호 품질 왜곡
- SKU 여부: [x] SKU — 업종(맛집/카페 vs 일반)별로 분화된 독립 상품. 단가 다름.

- 한 줄 정의: 검색 유입 → 체류 → 저장하기까지 3단계 행동을 하나의 상품으로 묶어 판매
- 번들 구성: 유입 + 체류 + 저장하기

- SKU 여부: [x] SKU — 2중 번들과 단가가 다르게 책정됨. 독립 상품 확인.

## 3. 검색 유입 리워드 (키워드 유형별) — 소소분류

### 3-a. 업종+지역 키워드 유입 리워드

- 키워드 유형: 대형 경쟁 키워드. 검색량 많으나 이미 경쟁 포화 상태.
- 발굴 방식: 수동 리서치 또는 사장님닷컴(sjnim.com) 키워드 도구 활용
- 출처 URL: https://adall.kr/blog/c78b3226-8281-4250-a35b-790d9761f9fd/ (키워드 유형 분류 명시), https://www.i-boss.co.kr/ab-2987-516347 (5~10위 키워드부터 접수 가능 구조 확인)

- SKU 여부: [x] SKU — 키워드 난이도(대형/중형)에 따라 단가가 다르게 책정됨. 독립 상품.

### 3-b. 메뉴명/서비스명 키워드 유입 리워드

- 발굴 방식: 매장 메뉴 기반 수동 설정
- 출처 URL: https://adall.kr/blog/c78b3226-8281-4250-a35b-790d9761f9fd/ (메뉴명/서비스명 키워드 분류 명시)
