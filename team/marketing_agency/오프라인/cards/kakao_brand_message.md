### 대행료 구조

- 에이전시 대행료 책정 방식: 발송 건수 기반 수수료 또는 월 고정 운영비 혼합 구조가 일반적
- 크몽 공개 "카카오 브랜드메시지 발송 대행" 가격표: 미확정 유지 — 크몽 검색 결과는 비공식 PC카톡 매크로 프로그램 위주. 정식 채널 기반 대행 서비스 공개 가격표 없음
- 카카오 공식 통합에이전시 수수료: 실적정산금액(유상캐시 소진분) × 수수료율 구조. 수수료율 비공개. 월 15일 정산. (출처: kakaobusiness.gitbook.io/agency_guide)
- 일반 대행 에이전시 월 운영 수수료: 미확정 — 블로그/공개 가격표 없음. 별도 문의 필요

### 결제 구조

- 발송 비용: 발송 전 플랫폼 충전금 선불 구조 (비즈고, 솔라피, 알리고 공통)
- 대행료: 월 선금 또는 캠페인 선금 방식이 일반적
- 부가세: 10% 별도

## B. 실행

### 타입별 실행 구조

**F 타입 (진입 장벽 최저 — 자영업자 기본 진입 경로)**
- 발송 대상: 카카오 채널을 추가한 친구 전원. 마케팅 수신 동의 여부 무관.
- 수신번호 불필요. 채널 보유만으로 즉시 진입 가능.
- 적합 시나리오: 휴무 안내, 이벤트 공지, 시즌 이벤트 쿠폰, 일괄 쿠폰 배포
- 지원 플랫폼: 비즈고, 솔라피, 알리고

**I 타입 (구 친구톡 전환 경로 — 진입 조건 있음)**
- 발송 대상: 채널 친구 + 마케팅 수신 동의 교집합. 최고 충성도 고객군.
- 자격 조건: 채널 친구 5만 이상, 알림톡 발송 이력, 080 수신거부 번호 등 (출처: https://blog.bizgo.io/howto/understanding-kakao-brand-message-targeting-types/)
- 적합 시나리오: VIP 혜택, 프리오더, 고가 상품 안내, 사전 예약
- 지원 플랫폼: 비즈고, 솔라피, 알리고, 인포뱅크

**M 타입 (진입 장벽 높음 — 대형 사업자 위주)**
- 발송 대상: 채널 친구 무관, 카카오톡 마케팅 수신 동의 이력 보유 고객 전체
- 자격 조건: 채널 친구 5만 이상 + 알림톡 발송 이력 + 080 수신거부 번호 + 수신자 전화번호 DB 보유 필수
- 오프라인 자영업자 대부분이 이 DB 미보유 → 현실적 진입 어려움
- 지원 플랫폼: 비즈고, 솔라피, 인포뱅크

**N 타입 (신규 고객 유입 목적 — M 타입에서 비친구 추출)**
- 발송 대상: M 그룹 중 채널 친구가 아닌 사람. 채널 추가 유도 목적.
- 자격 조건: M 타입과 동일 조건 + 수신번호 보유
- 적합 시나리오: 웰컴 메시지, 채널 추가 유도 쿠폰, 첫 방문 후 재방문 전환
- 지원 플랫폼: 비즈고, 인포뱅크

### 딜러사 계약 필요 여부
- 브랜드 메시지 발송을 위해 카카오 딜러사(비즈고, 솔라피 등)와 계약 필요
- 대행사는 딜러사 계정을 통해 발송 대리 진행

### 리드타임
- F타입 텍스트형: 원고 확정 후 당일~1일 이내 발송 가능
- 이미지형/와이드형: 디자인 제작 포함 시 2~5일 소요 예상
- 캐러셀/자동화 설정: 미확정 — 재조사 필요

### 납품 형태
- 발송 완료 캡처 + 발송 건수 확인 리포트 제공
- 열람률·클릭률 리포팅: 플랫폼 지원 여부에 따라 다름 (미확정 — 재조사 필요)

### 고객이 제공해야 하는 것
- 카카오 채널 계정 접근 권한 또는 채널 정보
- 이벤트/프로모션 내용 및 소재 (이미지형 이상의 경우 이미지 원본 또는 제작 의뢰)
- M/N/I 타입의 경우 수신자 전화번호 DB (마케팅 수신 동의자 명단)

### 적합 업종
- 미용실, 네일샵, 피부관리실 (재방문 주기 관리)
- 카페, 음식점 (시즌 이벤트, 메뉴 소개)
- 학원, 필라테스, 요가 스튜디오 (수강생 공지)
- 소규모 편집샵, 의류점 (신상품 안내)
- 세탁소, 수선점 (운영 시간 변경 공지)

### 부적합 업종
- 수신자 DB가 없고 채널 친구도 없는 신규 개설 점포 (발송 대상 자체가 없음)
- 커머스형 SKU: 온라인 결제 연동이 없는 순수 오프라인 점포
- 캐러셀커머스형: 오프라인 전용 자영업자

### 발송 시간 실무 주의
- 오전 11시 발송이 자영업자 사이에서 집중. 점심 직전 접촉 최고 효율
- 20:50~익일 08:00 발송 금지 (법적 의무)
- 11월(빼빼로데이·수능)이 자영업자 쿠폰 발송 최성수기
- 출처: https://tossplace.com/story/crm_yearend2025

## C. 계약

### 최소 계약 기간
- 단발 캠페인형 (시즈널): 건별 계약 가능
- 월 정기 운영형: 3개월 이상 권장 (자동화 설정 초기 비용 회수 관점)
- 구체적 최소 기간 표준: 미확정 — 대행 에이전시별 상이

### 해지 조건
- 단발형: 발송 완료 후 자동 종료
- 월 정기형: 1개월 전 해지 통보 조건이 일반적 (업계 표준 — 계약서 필수 명시)

### KPI 정의
- 기본 KPI: 발송 완료 건수 (100% 달성 기준)
- 선택 KPI: 열람률, 클릭률 (플랫폼 제공 데이터 기준)
- 전환 KPI (방문, 구매): 측정 방법 계약 시 명시 필요. 미측정 시 분쟁 원인이 됨
- 주의: 브랜드 메시지 자체는 발송까지만 대행. 전환 보장 불가를 계약서에 명시해야 함

### 보고 주기
- 단발형: 발송 완료 후 1회 보고
- 월 정기형: 월 1회 성과 보고 (발송 건수, 열람률) 권장

### 재작업 한도
- 원고 수정: 발송 전 1~2회 이내로 제한 명시 권장
- 발송 후 재발송: 발송 오류(플랫폼 장애) 외에는 원칙적으로 불가. 재발송 시 추가 발송 비용 발생

### 지적재산권
- 대행사가 제작한 이미지 소재: 계약서에 납품 후 클라이언트 귀속 명시 권장
- 원고 및 디자인 저작권 귀속 조건을 계약서에 명시

### 분쟁 예방 체크포인트
- 발송 대상 DB 제공 주체와 적법성 확인 서면 서명 (수신 동의 여부)
- 발송 건수 확인 캡처 납품 필수
- "발송만 대행, 효과 보장 아님" 문구 계약서 포함

## G. 리스크

### 스팸·불법 발송
- 2026년 전환 이후 템플릿 사전 승인 절차 폐지 → 규정 위반 책임이 발송자(대행사 포함)에게 귀속
- 정보통신망법상 야간 발송 금지(20:50~익일 08:00) 위반 시 과태료
- 허위·과장 내용 포함 시 표시광고법 위반 가능성
- 수신 동의 없는 DB에 M/N 타입 발송 시 개인정보보호법 위반

### 수신거부 처리
- 080 무료수신거부 번호 운영 의무 (M 타입 이상 조건 진입 시 필수)
- 수신거부 신청자 즉시 처리 의무. 처리 지연 시 법적 책임
- 대행사가 수신거부 처리를 클라이언트 대신 진행하는지 계약서 명시 필요

### 개인정보보호법
- M/N/I 타입 발송 시 수신자 전화번호 DB는 마케팅 수신 동의 이력이 있는 개인정보
- 클라이언트가 제공한 DB의 수신 동의 적법성 확인 책임 소재를 계약서에 명시
- 대행사가 수신자 DB를 보관·처리하는 경우 개인정보처리 위수탁 계약 별도 작성 필요

### 진입 장벽 미확인으로 인한 분쟁
- M/I 타입 상품 판매 후 클라이언트가 친구 수 5만 미달로 진입 불가 판명 시 환불 분쟁
- 사전 타입별 자격 조건 확인서 서명 필수
- F 타입 위주로 시작하고 조건 미달 시 F 타입 전환 옵션을 계약서에 명시

### DB 미보유 자영업자 공급 구조
- 오프라인 자영업자 대부분은 마케팅 수신 동의 DB 미보유
- M/N 타입 판매 시 DB 구축 과정(카카오채널 가입 동의 유도, 키오스크 연동 등)을 별도 서비스로 패키징하지 않으면 판매 후 이행 불가 상황 발생
- 자영업자 대상 현실적 진입 경로: F 타입 → DB 축적 → I/M 타입 단계적 확장

### 환불·클레임 빈발 구조
- "발송 완료했는데 손님이 안 왔다"는 클레임 — 전환 보장 불가 조건 계약서 명시 필수
- 플랫폼 장애로 발송 실패 시 플랫폼 책임 vs 대행사 책임 구분 명시 필요
- 발송 완료 증빙(건수 캡처, 발송 로그) 납품 필수

## DB 구축 문제 (자영업자 대상 대행 핵심 장벽)

오프라인 자영업자의 브랜드 메시지 대행에서 가장 큰 구조적 장벽.

- M/N 타입의 전제 조건인 "마케팅 수신 동의 DB"를 오프라인 자영업자 대부분이 미보유
- DB 확보 방법: 카카오 채널 가입 시 동의 유도 / 키오스크 연동 / 카카오페이 결제 시 동의 수집
- 이 과정을 대행하는 서비스가 업계에서 거의 공개되지 않음 — 차별화 기회 포인트
- 현실적 대행 구조: F 타입으로 시작 → DB 축적 → M/I 타입 확장 단계 제안
