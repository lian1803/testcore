### 리뷰 최신성 (Review Recency)

**핵심 원칙: 총 개수 → 최신성으로 가중치 이동 [검증]**

네이버는 리뷰 총 개수에서 "최근 리뷰가 얼마나 달렸는지"로 평가 기준을 이동했다. 배경은 구조적 문제: 총 개수 기준이었을 때 신규 업체가 상위 노출될 기회가 없었고 승자 독점 구조가 고착화됐다. 이를 해소하기 위해 최신성 기준이 도입됐다.

- "지역, 업종에 따라 일정 기간에 몇 개 이상이 되면 만점을 준다" — 즉 절대 총량이 아니라 최근 기간 내 누적 속도가 평가 기준 [검증]
- 출처: marketing-way.co.kr (리뷰 5가지 원칙)

**최근 기간 기준 — 며칠인가?**

- 정확한 기준 기간(30일/60일/90일 등) 공식 발표 없음 [알 수 없음 — 네이버 비공개]
- 커뮤니티에서 "최근 리뷰"라는 표현을 쓰나 구체적 윈도우(window)를 수치로 제시한 출처 없음
- 추론 근거: 네이버 플레이스 순위가 "기본 3개월 단위로 변동"한다는 언급이 있으므로, 90일 이내가 최신성 평가 윈도우일 가능성 있음 [추정]

**오래된 리뷰 1,000개 vs 최근 30일 리뷰 10개 — 어느 쪽이 유리한가?**

직접 비교 실험 데이터 없음 [알 수 없음]. 그러나 커뮤니티 다수 합의는 명확하다:
- "예전에는 총 개수가 중요할 때가 있었다. 지금은 최근 개수를 본다" [검증]
- "오래된 리뷰만 많은 가게는 순위가 자연스럽게 떨어질 수 있다" [검증]
- 결론: 최근 리뷰 10개가 3년 전 리뷰 1,000개보다 현재 알고리즘에서 유리할 가능성이 높다. 단, 교차 검증 가능한 단일 실험 데이터 없음.

**리뷰 감쇠(Decay) 메커니즘**

- "리뷰 감쇠"라는 명시적 메커니즘 공식 발표 없음 [알 수 없음 — 네이버 비공개]
- 직접적 가중치 감소보다는 **상대적 감쇠** 방식으로 작동하는 것으로 추정: 내 리뷰가 오래되는 동안 경쟁 업체가 최신 리뷰를 쌓으면 상대적 순위 하락 [추정]
- 실험 결과나 사례 데이터 없음

**리뷰 속도/분산 패턴이 핵심 [검증]**

- 짧은 기간 집중 증가 → 비정상 신호로 감지 (어뷰징 의심)
- 일정한 간격으로 분산된 증가 → 자연 데이터 신호
- 즉, "30일에 100개"보다 "매주 5-10개씩 꾸준히"가 알고리즘에 더 우호적
- 출처: i-boss.co.kr (2026 리뷰 검색노출 전략)

**2026년 4월 리뷰 개편 — 별점 도입 [공식]**

- 2026년 4월 6일부터 5점 척도 별점 기능 추가
- 도입 초기에는 데이터 안정성 확보를 위해 업체주/작성자만 볼 수 있는 수집 전용 기간 운영
- 리뷰 수정 가능 기간을 작성 후 3개월 이내로 제한 (어뷰징 방지)
- 업체 사장이 숫자 평점 노출 여부를 선택 가능
- 출처: gamevu.co.kr 보도 (2026.04)

**진단 시스템 적용 시사점**

- 최근 30일/90일 리뷰 수를 총 리뷰 수와 분리해서 추적하는 것이 유효
- 다만 정확한 평가 윈도우가 비공개이므로 30일/90일 두 구간 모두 수집 권장
- 리뷰 속도(월별 증가량) 추적 필드 추가 가치 있음

### 새소식 업데이트 타이밍

**업데이트 주기별 순위 차이 — 구체적 수치 없음 [알 수 없음]**

- 7일/14일/30일/90일 이내 각각의 순위 차이를 수치로 비교한 공식 실험 데이터 없음
- 네이버 비공개 정책으로 인해 커뮤니티에서도 구체적 임계값(threshold) 합의 없음

**"활성 업체" 판단 기준**

- 커뮤니티 권장 기준: 30일 이내 업데이트 = 활성 [검증]
- "자주 업데이트되는 가게가 더 관리 잘 되는 곳으로 인식된다" — 정보 최신성과 정확성을 활성 신호로 해석 [검증]
- 출처: storeartmagazine.com (정보 수정 관련 기사)
- 180일 이상 미업데이트 = "방치" 범주 [검증 — 커뮤니티 다수 합의], 그러나 공식 임계값 아님

**새소식 삭제 시 최신성 점수 하락 여부**

- 데이터 없음 [알 수 없음]. 게시물 삭제의 순위 영향에 대한 공개 실험 없음.

**"새소식 없음" vs "30일+ 된 새소식 있음" — 어느 쪽이 더 나쁜가?**

- 명확한 비교 데이터 없음 [알 수 없음]
- 논리적 추론: 오래된 새소식이 있다고 해서 패널티를 주기보다, 새소식 없는 업체와 동일한 "비활성" 취급을 받을 가능성 [추정]
- 실무 권장: 업로드 후 오래 방치하는 것보다 정기 업로드 유지가 안전

**새소식 업로드 후 순위 즉시 반영 여부**

- 즉시 반영 아님 [검증]
- 네이버 플레이스 순위가 2~3주 주기로 지역/업종별 서브 키워드 기준으로 갱신된다는 언급 있음 [추정 — 단일 출처]
- 출처: pinead.co.kr (업종별 전략 사례)
- 결론: 업로드 직후 순위 변화 없을 수 있으며, 반영까지 수 주 지연 가능

**최적 업데이트 주기**

- KB 가이드 권장: 매주 1회 [공식 수준 — KB 가이드]
- 커뮤니티 공통 권장: 월 2~4회 새소식 게시 [검증]
- 최소 기준: 월 1회 이상 [검증]

### 사진 업로드 시간성

**최근 N일 이내 사진 없을 시 최신성 점수 하락 여부**

- 공식 발표 없음 [알 수 없음 — 네이버 비공개]
- 사진 업로드가 직접적인 별도 최신성 지표로 존재하는지, 아니면 "활동 신호" 통합 지표의 일부인지 불명확

**사진 업로드의 알고리즘적 의미**

- "활성화된 매장으로 인식하게 해 노출 순위에 긍정적 영향" [검증]
- 사진 업로드 자체가 직접적 순위 상승 요소라기보다, **업체 활성 신호 복합 지표의 구성 요소**로 해석하는 것이 정확 [검증]
- "정기적인 사진 업로드 = 활성화 신호" 는 커뮤니티 다수 합의
- 권장 주기: 매주 1~2장 [검증]

**사진 날짜 메타데이터 활용 여부**

- 네이버가 사진 EXIF 메타데이터(촬영일 등)를 순위에 활용한다는 공식 발표 없음 [알 수 없음]
- 업로드 날짜(서버 수신 시점)는 내부적으로 기록되나 외부에서 확인 불가

**영상 콘텐츠 우선성 [검증]**

- 네이버는 영상 콘텐츠를 사진보다 우선 노출하는 경향 있음
- 새소식에 영상 업로드 시 대표 이미지 자리를 영상이 차지 — 의도하지 않으면 대표사진 노출 손실 가능
- 출처: storeartmagazine.com (대표사진 노출 기준 기사)

**답글 품질 [검증]**

### 종합: 시간 기반 활동지수 설계 제안

**"소통지수" 또는 "활동지수" 독립 지표 존재 여부**

- 네이버가 "소통지수"나 "활동지수"를 공개적으로 명명한 사례 없음 [알 수 없음 — 네이버 비공개]
- 1차 리서치에서 추출한 3대 축(적합도/인기도/신뢰도) 외에 "최신성" 또는 "활동성"이 별도 4번째 축으로 존재한다는 커뮤니티 추론 있음 [추정]
- "활동성 + 최신성 + 반응도를 보는 추세" — 명확한 지표명 없이 이 3가지가 복합적으로 작용 [검증]
- 2025년 5월 알고리즘 개편 이후 핵심 신호는 실시간 고객 행동 패턴(전화/예약/길찾기/체류/재방문)으로 이동 [검증]

**업체 로그인/관리 이력의 순위 영향**

- 직접적 로그인 빈도를 순위 신호로 쓴다는 공식 발표 없음 [알 수 없음]
- 간접 경로: 로그인 없으면 정보 업데이트 안 됨 → 정보 최신성 하락 → 순위 하락 [검증]
- 실질적으로 관리=활동 신호이므로 로그인과 순위는 간접 상관관계

**점수 설계 제안 (시간 기반 활동지수)**

1차 리서치 기획안의 새소식 최신성 구간(30/90/180일)은 커뮤니티 검증 기반으로 유효하다. 여기에 추가:

```
리뷰 최신성 점수 (시안):
- 최근 30일 리뷰 3개+: 만점 (지역/업종별 기준 미공개이므로 보수적 적용)
- 최근 90일 리뷰 5개+: 상위 구간
- 최근 90일 리뷰 0개: 최신성 감점 구간

리뷰 속도 보너스:
- 월 평균 증가율 측정 → 꾸준히 증가 = 보너스 / 정체 = 패널티 없으나 경고
```

**핵심 설계 원칙 3가지**

1. 시간 기반 신호는 "있다/없다"보다 **"얼마나 최근에, 얼마나 자주"**가 핵심 — 모든 항목에 날짜 기반 값 병행 수집 필요
2. 정확한 임계값은 네이버 비공개이므로 점수 구간은 **커뮤니티 다수 합의 기반 + 보수적 적용** 원칙으로 설정
3. 시간 기반 활동지수 = 새소식 + 사진 + 리뷰 속도 + 정보 수정 빈도를 합산한 단일 점수로 통합 관리하되, 각 항목 개별 표시도 병행

### 데이터 한계 (2차 리서치)

- 리뷰 최신성 평가 윈도우(정확히 며칠) — 네이버 비공개, 커뮤니티에서도 수치 합의 없음
- 리뷰 감쇠(decay) 메커니즘 — 직접 가중치 감소인지 상대적 감쇠인지 불명확
- 새소식 업로드 후 순위 반영까지의 지연 시간 — 2~3주 추정치 1개 출처, 교차 검증 없음
- 사진 날짜 메타데이터 활용 여부 — 미확인
- 리뷰 답글 속도의 알고리즘 신호 여부 — 미확인 (톡톡 응답 속도와 동일 취급인지 불명확)
- 정보 수정 주 5회 제한 — 단일 출처, 교차 검증 필요
- "소통지수/활동지수" 공식 명칭 — 존재하지 않거나 비공개

### 사진 기준

**공식 규격 [공식]**
- 최소 해상도: 160px × 160px 이상
- 권장 해상도: 가로 1,300px 이상 (마케팅 전문가 권장: 1,920×1,080 이상)

- 파일 용량: 최대 20MB (실용 권장: 2MB 이하)
- 허용 포맷: JPG, JPEG, GIF, PNG
- 출처: 네이버 스마트플레이스 공식 사양 (검색 결과 다수 일치)

**권장 사진 수 [검증]**

  | 업종 | 실용 최소 | 커뮤니티 권장 |
  |---|---|---|
  | 요식업 | 10장 | 20~30장 |

**리뷰 수 구간별 [추정]**
- 리뷰 총 개수 절대값보다 "최근 리뷰 증가 흐름"이 더 중요 (2025~2026 알고리즘)
- 구간별 순위 변화 공개 데이터 없음 — 진단 시스템 기획안 수치(0/12/24/32/40점 구간)는 커뮤니티 다수 검증 기반이나 공식 아님
- 커뮤니티 실무 기준: 리뷰 30개+ = 상위 노출 경쟁권 진입, 50개+ = 안정권

**리뷰 유형별 가중치 [검증]**
- 영수증리뷰(방문자리뷰) > 예약자리뷰 > 블로그리뷰 순으로 신뢰도 가중치
- 영수증리뷰 vs 블로그리뷰: 영수증리뷰가 조회수 약 10배 이상 기록 (가중치 간접 지표)
- 2025년 알고리즘 개편 이후 영수증리뷰 비중 강화 — "실제 방문 인증" 처리
- 출처: j-stagram.com 영수증/방문자리뷰 가이드

**사장님 댓글(Owner Reply) [검증]**
- 답글 자체가 직접 순위 올리는 것 아님
- 답글 → 고객 체류시간 증가 → 간접적 순위 상승 경로
- 답글 내 지역명 포함 권장 (로컬 SEO 신호)
- 공식 응답률 수치 기준 미공개
- 커뮤니티 권장: 최근 리뷰 기준 80% 이상 응답
- 출처: marketing-way.co.kr, i-boss.co.kr

**리뷰 키워드 [검증]**
- 리뷰에 포함된 키워드가 검색 노출에 영향
- 리뷰 작성자가 쓴 키워드 = 해당 키워드 검색 시 노출 가능성 상승
- 리뷰 집중 생성(하루에 몰아서) 어뷰징 감지 위험 — 분산 생성 권장

**별점/감성 [추정]**
- 별점 평균이 노출 순위에 반영된다는 공식 발표 없음
- 다만 낮은 별점 = 신뢰도 하락 = 인기도 지표 하락 간접 경로
- 별점 4.0 이하 구간은 CTR 하락으로 인한 순위 하락 가능성

### 핵심 결론 — 진단 시스템 적용 포인트

1. **사진**: 10장 최소 기준 + 매주 1~2장 업데이트가 핵심. 해상도 1,300px+ 권장.
2. **소개글**: 300자 미만은 의미 없음. 500자+ 권장, 2,000자 최대 활용.

5. **솔루션**: 예약 연동이 가장 강력 (페이지뷰 3.1배). 톡톡(2.2배), 스마트콜(1.2배) 순.

## 지역 벤치마크 크롤러 기획 (2026-04-11)

### 왜 필요한가
- 웹 리서치 한계: 네이버 알고리즘 가중치 비공개 → 커뮤니티 추정치만 존재
- **해결책**: 우리 크롤러로 타겟 지역 상위 10개 업체 직접 수집 → 진짜 로컬 벤치마크 확보
- "양주 미용실 상위 10% = 리뷰 47개, 사진 23장" 같은 실측치 → 커뮤니티 추정보다 100배 정확

### 구현 계획
- `services/local_benchmark.py` 신규 생성
- 입력: category + 지역명 (예: "미용실", "양주")
- 동작: 해당 키워드 검색 → 상위 10개 업체 순서대로 크롤링
- 출력: `{ "top10_avg_review": 47, "top10_avg_photo": 23, "top3_avg_review": 89, ... }`
- 활용: DiagnosisScorer에서 competitor_avg_* 폴백 대신 실측치 사용

### 수집 항목 (업체당)
- 리뷰 수 (방문자+영수증)
- 사진 수
- 블로그 리뷰 수
- 새소식 유무 + 최신성
- 편의기능 (예약/톡톡/쿠폰)
- 소개글 유무

### 실행 방식
- 배치 진단 시작 전 해당 지역+업종 벤치마크 1회 수집 (캐싱)
- 캐시 유효기간: 7일 (자주 변하지 않음)
- 결과를 `config/industry_weights.py`의 폴백값 대신 실측값으로 자동 교체
