# CDO 분석 — 소상공인 네이버 플레이스 영업툴 업그레이드

---

## 디자인 비전

"진단 완료 → 메시지 복사 → 카톡 붙여넣기"가 60초 안에 끝나는, 영업 현장에 최적화된 내부 업무 툴.

비주얼 톤: 깔끔하고 전문적인 대시보드 감성. 화려함 없음. 숫자와 버튼이 주인공.
색감: 네이버 그린(#03C75A) 기반 신뢰감 + 카카오 옐로우(#FEE500) 액션 강조.
분위기: "공식 업무 툴처럼 보여야 한다" — 리안 자신이 쓰는 도구이고, PPT를 사장님에게 보낼 때 이 툴에서 나온 데이터가 신뢰감 있게 보여야 한다.

---

## 마케팅 채널 반영

채널 판단: 카카오 콜드DM + 문자(SMS) 병행.

UI 설계 방향:
- 메시지 카드가 화면의 중심이다. 점수나 그래프는 보조.
- 복사 버튼이 가장 크고 눈에 띄어야 한다. 리안의 핵심 액션은 "복사"다.
- 1차~4차 메시지가 탭으로 구분되어 있어서 "지금 어느 단계인지" 한눈에 파악 가능해야 한다.
- 카톡에 붙여넣었을 때 어색하지 않도록 메시지 미리보기가 실제 카톡 말풍선처럼 보여야 한다.
- 50대 이상 사장님 대상 문자용(SMS) 1차 메시지는 별도 표시 (짧고 간결함 강조).

---

## UX 원칙 (이 프로젝트 맞춤)

1. 사용자는 리안 혼자다. 온보딩 불필요. 속도가 전부.
2. 한 화면에서 "다음 액션"이 항상 명확해야 한다. 고민할 게 없어야 한다.
3. 메시지 복사가 0클릭에 가까워야 한다. 결과 화면 열리자마자 1차 메시지가 바로 보여야 한다.
4. 점수/등급은 "왜 연락해야 하는가"를 설명하는 근거 데이터. 시각적으로 정리하되, 공간을 너무 많이 차지하면 안 된다.
5. 에러나 로딩 상태를 명확히 표시해라. 크롤링이 실패했을 때 리안이 당황하지 않게.

## 핵심 화면 흐름

```
index.html (업체명 검색)
    ↓ 검색 실행
select.html (업체 선택 — 동명 업체 2개 이상일 때)
    ↓ 업체 선택
loading.html (크롤링 진행 중)
    ↓ 완료
result.html [업그레이드] (진단 결과 + 영업 메시지 1~4차)
    ↓ 이력 보기
history.html [업그레이드] (이력 목록 + 우선순위 필터)
```

단건 플로우가 핵심. batch.html은 이번 MVP 제외 (CPO 결정 반영).

---

## 각 화면 설계

---

### 화면 1: index.html (메인/검색)

목적: 업체명 입력하고 진단 시작.

현재 구조 유지. 개선 포인트만:
- 검색창이 화면 중앙에 크게. 주변 여백 충분히.
- 엔터키로도 검색 실행 (현재 버튼 클릭만 가능하다면 추가).
- 최근 진단 업체 3개를 검색창 아래에 작게 표시 (클릭하면 해당 result.html로 바로 이동).
  - 이유: 리안이 어제 진단한 업체에 오늘 2차 메시지 보낼 때, 다시 검색하는 게 아니라 바로 접근할 수 있어야 한다.
- CTA: "진단 시작" 버튼 — Primary 컬러 (네이버 그린).

---

### 화면 2: select.html (업체 선택)

목적: 동명 업체 중 정확한 업체 선택.

현재 구조 유지. 개선 포인트만:
- 업체 카드에 주소 + 업종 + 네이버 플레이스 썸네일(있으면) 표시.
- 선택 즉시 loading으로 이동 (클릭 → 확인 버튼 없이 바로).

---

### 화면 3: loading.html (크롤링 진행 중)

목적: 기다리는 동안 리안이 다른 행동을 안 하도록.

개선 포인트:
- 단계별 진행 표시: "플레이스 검색 중..." → "상세 데이터 수집 중..." → "점수 계산 중..." → "메시지 생성 중..."
- 진행 단계가 텍스트로 바뀌면서 진행 중임을 인식하게.
- 예상 소요 시간 표시: "약 20~30초 소요".
- 오류 발생 시: 빨간 배경 없이 "크롤링 실패 — 수동 입력으로 진행하시겠습니까?" + 두 개 버튼 (재시도 / 수동 입력).

---

### 화면 4: result.html [핵심 업그레이드]

목적: 진단 결과 확인 + 1~4차 메시지 복사.

#### 전체 레이아웃 (위에서 아래로)

```
[상단 헤더 바]
업체명 + 등급 배지 + 종합 점수 + 우선순위 태그 + [PPT 다운로드 버튼] [이력으로 버튼]

[진단 요약 섹션] — 접을 수 있는 아코디언
7개 항목 점수 바 차트 (가로 막대) + 핵심 수치 3개 강조

[영업 메시지 섹션] — 항상 펼쳐진 상태, 화면 아래 고정 영역 아님
탭: [1차 메시지] [2차 메시지] [3차 메시지] [4차 메시지]
메시지 미리보기 카드 + 복사 버튼
```

#### 상단 헤더 바 상세

```
[업체명 (크게)] [등급 배지: A/B/C/D 색상 구분] [종합점수 숫자]
[우선순위 태그: 1순위 (빨강) / 2순위 (주황) / 패스 (회색)]
                                    [PPT 다운로드] [이력 보기]
```

- 등급 색상: A = 초록, B = 파랑, C = 주황, D = 빨강.
- 우선순위 태그는 자동 분류 결과. 리안이 수동으로 변경 가능 (태그 클릭 → 드롭다운).

#### 진단 요약 섹션

기본 상태: 접혀 있음. "진단 상세 보기 ▼" 클릭해서 펼침.
이유: 리안의 목적은 메시지 복사다. 점수는 필요할 때만 본다.

펼쳤을 때 표시:
- 가로 막대 차트 7개 항목 (각 항목명 + 점수/100 + 막대 길이).
- 핵심 수치 3개 하이라이트 카드:
  - 리뷰 수 (경쟁사 평균 대비 몇 개 부족한지)
  - 현재 순위 (검색 시 몇 위에 노출되는지)
  - 손실 고객 추정 (월 N명 경쟁사로)

핵심 수치 카드 디자인:
```
┌─────────────────────────────┐
│  리뷰 부족                   │
│  경쟁사 평균보다 37개 적음     │
│  (내 리뷰 8개 / 이 동네 평균 45개) │
└─────────────────────────────┘
```
숫자는 크게. 설명은 작게. 배경은 연한 경고색 (연한 주황 또는 연한 빨강).

#### 영업 메시지 섹션 — 최우선

탭 구조:

```
[1차 메시지 ●] [2차 메시지] [3차 메시지] [4차 메시지]
```

탭에 숫자 뱃지로 "몇 개 버전이 있는지" 표시. 1차는 A/B/C 버전이므로 탭에 "1차 (3개)" 표시.

**1차 메시지 탭 내부:**

```
자동 선택된 버전 표시:
[● A버전 — 리뷰 격차형] [ B버전 — 손실 고객형] [ C버전 — 방치형]
(자동 선택 이유: "경쟁사 리뷰가 내 리뷰의 5배 이상 → A버전 선택")

─────────────────────────────────────────────
메시지 미리보기 (카톡 말풍선 스타일):

  안녕하세요! [업체명] 사장님 :)
  네이버에서 [업체명] 검색해보니
  근처 경쟁 업체보다 리뷰가 37개 적더라고요.

  혹시 무료 플레이스 진단 보내드려도 될까요?

─────────────────────────────────────────────

[카카오용 복사]   [SMS용 복사 (짧게)]
```

- 말풍선 배경: 연한 회색 (카톡 말풍선과 유사한 느낌).
- "카카오용 복사" 버튼: 카카오 옐로우 (#FEE500) 배경, 검은 텍스트.
- "SMS용 복사" 버튼: 회색 배경 (50대+ 대상, 문자 발송용).
- 복사 성공 시: 버튼 텍스트가 "복사됨 ✓"으로 1.5초 변경 후 원복.

**2차, 3차, 4차 메시지 탭 내부:**

동일한 구조. 단, 2차~4차는 버전 선택 없이 단일 메시지.

4차 메시지는 3개 버전 (보류/비싸다/직접하겠다) 탭 내 라디오 버튼으로 선택.

**메시지 수동 편집 기능:**

각 메시지 우측 하단에 "편집" 버튼. 클릭하면 텍스트에어리어로 전환.
편집 후 "완료" → 편집된 내용이 복사 대상이 됨.
"원본으로 복구" 버튼 별도 제공.

---

### 화면 5: history.html [업그레이드]

목적: 이력 목록 조회 + 우선순위 필터링.

#### 레이아웃

```
[상단 필터 바]
[전체] [1순위] [2순위] [패스] | 업종 드롭다운 | 날짜 정렬

[업체 카드 목록] (세로 스크롤)
```

#### 업체 카드 설계

```
┌──────────────────────────────────────────────────────────┐
│ [우선순위 태그: 1순위]  홍길동미용실    D등급  47점        │
│ 의정부시 가능동 | 미용실 | 진단일: 2026-03-26             │
│ 리뷰 8개 · 사진 5장 · 순위 15위                          │
│                               [결과 보기] [메시지 보기]   │
└──────────────────────────────────────────────────────────┘
```

- 우선순위 태그 색상: 1순위 = 빨강, 2순위 = 주황, 패스 = 회색.
- "메시지 보기" 버튼: result.html의 메시지 섹션으로 직접 링크 (진단 재실행 없이).

#### 필터 동작

- 탭 클릭 즉시 필터링 (페이지 리로드 없이 JS로).
- 필터 상태에서 업체 수 표시: "1순위 (12개)".

## 컴포넌트 시스템

### 컬러 팔레트

| 용도 | 색상 | 코드 |
|------|------|------|
| Primary (브랜드/신뢰) | 네이버 그린 | #03C75A |
| Secondary (구조/배경) | 라이트 그레이 | #F5F5F5 |
| Accent (카카오 CTA) | 카카오 옐로우 | #FEE500 |
| 경고/1순위 | 빨강 | #FF4757 |
| 2순위 | 주황 | #FFA502 |
| 패스 | 미드 그레이 | #A4A4A4 |
| 텍스트 기본 | 다크 그레이 | #2D2D2D |
| 텍스트 보조 | 미드 그레이 | #6B6B6B |

### 타이포그래피

- 제목 (업체명, 섹션 헤더): 700 굵기, 18~24px.
- 본문 (점수, 설명 텍스트): 400 굵기, 14~16px.
- 메시지 텍스트: 400 굵기, 15px, 행간 1.6 (카톡 가독성 기준).
- 강조 숫자 (손실 고객 수, 리뷰 격차): 700 굵기, 28~36px, Primary 또는 경고색.

### 폰트

- 시스템 폰트 우선: `-apple-system, BlinkMacSystemFont, 'Noto Sans KR', sans-serif`.
- 추가 로드 없이 빠른 렌더링.

### 핵심 컴포넌트 목록

shadcn/ui 사용 불가 (Jinja2 + Vanilla JS 스택). 동일한 디자인 패턴을 Tailwind CSS 클래스로 직접 구현.

| 컴포넌트 | 구현 방식 |
|---------|----------|
| 등급 배지 | `<span>` + Tailwind 색상 클래스 |
| 우선순위 태그 | `<span>` + 색상 + 클릭 시 드롭다운 |
| 가로 막대 차트 | CSS `width` 애니메이션 (JS 없이) |
| 메시지 미리보기 카드 | `<div>` 말풍선 스타일 (border-radius + 배경) |
| 탭 (1~4차) | Vanilla JS 탭 전환 |
| 복사 버튼 상태 변경 | JS `setTimeout` + 클래스 토글 |
| 아코디언 (진단 상세) | HTML `<details>/<summary>` 네이티브 사용 |
| 필터 탭 (history) | JS 필터링 + 카운트 업데이트 |

## 전환 최적화 포인트

리안이 영업 툴을 "빠르게 쓰는 것"이 이 툴의 전환 목표.

1. result.html 진입 시 자동 스크롤 없이 1차 메시지 탭이 기본 선택 상태.
   - 이유: 진단 완료 직후 리안이 바로 복사할 수 있어야 함.

2. 복사 버튼은 항상 화면에서 가장 눈에 띄는 요소.
   - 카카오 옐로우 배경 사용. 다른 버튼보다 1.5배 이상 크게.

3. PPT 다운로드 버튼은 복사 버튼보다 작게, 헤더에 위치.
   - 이유: PPT는 2차 이후 미팅용. 1차 DM 단계에서는 메시지가 우선.

4. "편집" 기능은 눈에 덜 띄게 (작은 텍스트 버튼).
   - 이유: 대부분 자동 생성 메시지를 그대로 쓴다. 편집은 예외 케이스.

5. history.html에서 "메시지 보기" 버튼을 "결과 보기"보다 오른쪽에 강조 배치.
   - 이유: 이전에 진단한 업체에 2~4차 메시지 보내러 오는 게 주요 재방문 목적.

6. 로딩 화면의 단계 표시 = 이탈 방지.
   - "20~30초 소요" 문구로 리안이 창을 닫지 않게.

## 점수 시각화 개선안

### 현재 문제
텍스트 기반 점수 나열 → 어느 항목이 약한지 한눈에 안 들어옴.

### 개선 방향

**1. 가로 막대 차트 (권장)**

```
사진       ████████░░░░░░░░  45/100
리뷰       ████░░░░░░░░░░░░  22/100  ← 빨강 (위험)
블로그     ██████░░░░░░░░░░  35/100
정보완성도  ████████████░░░░  72/100
키워드     ██████████░░░░░░  60/100
편의기능   ████████████████  100/100
참여도     ██████░░░░░░░░░░  38/100
```

- 60점 미만 항목 막대 색상: 빨강 또는 주황.
- 60점 이상: 그린.
- 애니메이션: 페이지 로드 시 0에서 채워지는 효과 (CSS transition).

**2. 핵심 수치 3개 카드 (별도 강조)**

점수 차트 위에 배치:

## CTO에게 요청

### 기술 확인 필요 사항

1. **Tailwind CSS 현재 포함 여부**: 기존 HTML 템플릿이 Tailwind를 사용하는지, 아니면 자체 CSS인지 확인. Tailwind가 없으면 CDN으로 추가 (개발용 CDN이면 프로덕션 전 빌드 필요).

2. **메시지 생성 결과를 result.html에 전달하는 방법**: Jinja2 템플릿이므로 서버사이드 렌더링. 메시지 4개 세트를 result.html 컨텍스트에 함께 넘겨주면 되는지, 아니면 result.html에서 별도 API 호출(AJAX)로 가져오는 방식인지 아키텍처 판단 필요.
   - CDO 선호: 서버사이드 렌더링으로 한 번에 전달. 로딩 화면 이후 result.html 렌더링 시 메시지도 함께. AJAX 별도 호출 불필요.

3. **탭 전환 Vanilla JS 구현**: shadcn 없이 Vanilla JS로 탭 구현. 기존 HTML 파일에 인라인 `<script>` 방식인지, 별도 static JS 파일로 분리 가능한지 확인.

4. **클립보드 복사 API 호환성**: `navigator.clipboard.writeText()` 사용. HTTPS 또는 localhost에서만 동작. 로컬 환경(http://localhost)이면 OK. 배포 환경이 HTTP이면 `document.execCommand('copy')` 폴백 필요.

5. **history.html 필터링 방식**: 현재 history.html이 서버에서 전체 목록을 렌더링하는 구조라면, 클라이언트사이드 필터링(JS로 DOM 숨기기/보이기)이 가장 빠름. DB 쿼리 필터링 방식(페이지 리로드)은 UX상 느림. 어느 방식이 기존 코드와 자연스럽게 맞는지 확인.

6. **우선순위 태그 수동 변경 저장**: 리안이 태그를 수동으로 바꾸면 DB에 저장돼야 함. PATCH `/api/businesses/{id}/priority` 엔드포인트 신규 추가 필요. 확인 요청.
