## 핵심 목표

리안이 소상공인에게 콜드DM으로 월 5건 계약을 달성하는 "진단-제안-클로징" 자동화 엔진.

**업체명 입력 → 60초 안에 진단 완료 + 1~4차 메시지 복붙 가능 상태**

---

## 사용자

- **사용자**: 리안 (1인 영업자, 경기북부 소상공인 대상 네이버 플레이스 마케팅 대행 영업)
- **영업 지역**: 경기북부 (양주, 포천, 의정부 등)
- **수익 모델**: 월 구독 29만~89만원
- **월 목표**: 5건 계약

## 현재 상태 (기존 코드 — naver-diagnosis/)

### 작동하는 것
- Playwright 기반 네이버 플레이스 크롤링
- 7개 항목 점수 체계 (사진/리뷰/블로그/정보/키워드/편의/참여)
- SQLite DB 저장 + 이력 관리
- PPT 생성 (7슬라이드, 데이터 나열 수준)
- Jinja2 웹 UI (index/result/history/batch/manual 화면)
- 네이버 검색광고 API (키워드 검색량)

### 문제점
- 점수가 절대값 기반 (경쟁사 대비 상대점수 없음)
- PPT가 리포트 수준 (영업 제안서가 아님, 충격/설득 없음)
- 메시지 생성 기능 없음 (1~4차 퍼널이 코드에 없음)
- 영업 우선순위 자동 분류 없음
- xlsx 배치 처리 없음

## Must Have (이번 업그레이드 범위)

### 1. 영업 메시지 자동 생성 [최우선]

진단 완료 후 결과 화면에 1~4차 메시지 탭 추가. 각 메시지 "복사" 버튼 (클립보드 복사).

**1차 메시지 — 업체 상황에 따라 A/B/C 자동 선택:**
- [A] 리뷰 격차형: 경쟁사 리뷰 >= 내 리뷰 × 5
- [B] 손실 고객형: 순위 > 10 AND 월검색 >= 500
- [C] 방치형: 새소식 >= 90일 OR 저장수 = 0

**2차 메시지**: 진단 데이터 → 카톡 카드 형식 텍스트 자동 완성

**3차 메시지**: 업종별 객단가 기반 손익분기 + 패키지 추천 자동 주입
- 패키지: 주목 290만원/월 / 집중 490만원/월 / 시선 890만원/월
- 업종별 객단가: 미용실 5~8만원 / 식당 1.5~3만원 / 피부관리 8~15만원 / 학원 20~40만원

**4차 메시지**: 보류/비싸다/직접하겠다 3가지 대응 버전 자동 생성

**구현 위치**: `services/message_generator.py` (신규) + `routers/message.py` (신규) + result.html 확장

---

### 2. PPT 영업 제안서화 [필수]

"문제는 확실히 보여주되, 해결법은 절대 안 알려준다"

**보여줘야 할 것 (공포/충격):**
- 표지: "사장님, 지금 매달 약 N명이 경쟁사로 가고 있어요"
- 경쟁사 vs 내 업체 숫자 격차
- 항목별 X 표시 목록

**절대 안 보여줄 것 (계약 후에만):**
- 구체적 개선 방법
- 경쟁사가 뭘 잘했는지

**슬라이드 구성:**
1. 표지 — 충격 문구 강화
2. 분석 결과 — 항목별 점수 (시각적 충격)
3. 세부 체크리스트
4. 개선사항 (방향만, 방법은 없음)
5. 키워드 분석
6. 연관 키워드
7. 손익분기 (신규) — 패키지별 "고객 N명만 더 오면 본전"
8. 마케팅 제안 (패키지 소개)

**손실 고객 수 계산**: 월 키워드 검색량 × (1 - CTR)
- CTR 기준: 1위 35%, 2위 17%, 3위 12%, 4~10위 4~8%, 10위+ 1~2%

**구현 위치**: `services/ppt_generator.py` 수정

---

### 3. 영업 우선순위 자동 분류 [필수]

진단 완료 시 자동 태그: **1순위(핫) / 2순위(웜) / 패스**

분류 기준:
- 1순위: D등급 + (사진<10 OR 리뷰<10 OR 순위>10 OR 블로그<5) 중 2개 이상
- 2순위: C등급 + 경쟁사 대비 절반 미만
- 패스: A등급, 프랜차이즈 본사 직영, 최신 리뷰 1년+ 된 업체

history.html에서 우선순위별 필터링/정렬.

**구현 위치**: `services/scorer.py` 확장 + `models.py`에 `sales_priority` 필드 추가

---

### 4. xlsx 연동 배치 처리 [Must Have — 010수집 연동]

`소상공인 010수집 최종본/db/` xlsx 파일 → 업체 목록 파싱 → 일괄 진단

- xlsx 업로드 또는 파일 경로 지정 → 업체명 컬럼 자동 감지
- 일괄 진단 큐 등록 → 진행 상황 실시간 표시
- 완료된 업체에 영업 우선순위 태그 자동 부여
- 업체별 결과 목록 UI (1업체 = 1행, PPT 다운로드 버튼)

**구현 위치**: `routers/batch.py` 확장 + `batch.html` 업그레이드
