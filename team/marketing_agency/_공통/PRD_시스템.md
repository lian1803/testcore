# 마케팅 에이전시 운영 시스템 PRD

> Product Requirement Document — 마케팅 에이전시 비즈니스 운영을 위한 통합 시스템

---

## 0. 요약

마케팅 에이전시가 100개 이상의 클라이언트를 효율적으로 관리하고, 패키지 제안 → 계약 → 온보딩 → 운영 → 성과 보고 → 갱신/종료까지의 전체 라이프사이클을 자동화할 수 있는 통합 플랫폼.

**핵심 가치**:
- 신규 클라이언트 획득 → 계약까지 자동 제안서 생성 (수동 작업 70% 감소)
- 광고 계정 + SNS + 분석 도구 통합 연동 (3개 플랫폼 접근 시간 60% 감소)
- 월별 성과 리포트 자동 생성 (리포팅 시간 80% 감소)
- 클라이언트 포탈을 통한 실시간 성과 추적 (만족도 +30%)

---

## 1. 프로젝트 개요

### 1-1. 배경 및 필요성

현재 마케팅 에이전시 운영의 문제점:

1. **클라이언트 관리의 비효율**
   - 스프레드시트로 클라이언트 정보 분산 관리
   - 제안서 매번 수동 작성 (반복 작업 70%)
   - 계약서 및 SOW 수동 작성 및 관리

2. **서비스 운영의 복잡성**
   - 광고 계정 (메타, 구글, 네이버 등) 개별 접근
   - 분석 도구 별도 연동
   - 성과 데이터 수동 수집 및 정리
   - 월별 리포트 매번 처음부터 제작

3. **팀 협업의 어려움**
   - 다양한 도구 (Slack, Google Drive, Notion, 스프레드시트) 분산 사용
   - 정보 일관성 부족
   - 진행 상황 추적 어려움

4. **클라이언트 경험 개선의 한계**
   - 성과 데이터 월 1회만 공개
   - 실시간 성과 조회 불가
   - 자동화된 보고 체계 없음

### 1-2. 해결 방안

통합 마케팅 에이전시 운영 시스템 구축:

1. **CRM (고객 관리 시스템)**
   - 클라이언트 정보 중앙화
   - 계약 생명주기 관리
   - 자동 제안서/계약서 생성

2. **서비스 패키지 카탈로그**
   - 10개 서비스별 3단계 패키지 (30개 조합)
   - 패키지 선택 → 자동 제안서 생성
   - 클라이언트별 커스터마이징 옵션

3. **통합 분석 대시보드**
   - 광고 계정 (메타, 구글, 네이버, 카카오, 틱톡) 자동 연동
   - 실시간 성과 데이터 수집
   - 클라이언트별 커스텀 대시보드

4. **자동 리포팅 엔진**
   - 월별 성과 리포트 자동 생성
   - PDF/PPT 자동 출력
   - 이메일 자동 발송

5. **클라이언트 포탈**
   - 실시간 성과 조회
   - 계약서, SOW, 리포트 다운로드
   - 티켓 시스템 (문의 접수)

## 2. 시스템 구성 및 화면 목록

### 2-1. 핵심 화면 (필수)

#### 1. 클라이언트 관리 (CRM)
**목적**: 모든 클라이언트 정보 중앙화 관리

**주요 기능**:
- [ ] 클라이언트 리스트 (검색/필터/정렬)
- [ ] 클라이언트 상세 정보 (회사명, 담당자, 연락처, 계약 정보)
- [ ] 계약 생명주기 상태 (계약 전 → 계약 → 진행 중 → 만료)
- [ ] 계약 갱신 알림 (60일 전)
- [ ] 클라이언트별 활동 로그 (미팅, 이메일, 파일)

**화면 요소**:
```
┌─────────────────────────────────────────┐
│ 클라이언트 관리 CRM                       │
├─────────────────────────────────────────┤
│ [검색 ] [필터 ▼] [추가 +]                  │
├─────────────────────────────────────────┤
│ 클라이언트명 │ 담당자 │ 상태 │ 계약금 │ 액션 │
├─────────────────────────────────────────┤
│ 홍길동 쇼핑몰 │ 김OO │ 진행중 │ 150만원 │ [보기]│
│ ...          │      │       │        │     │
└─────────────────────────────────────────┘
```

**기술 요구사항**:
- 데이터베이스: PostgreSQL (클라이언트 테이블)
- 인증: 직원별 권한 (관리자/PM/팀원)
- 통합: Slack 알림 (갱신 90일 전)

#### 2. 서비스 패키지 카탈로그
**목적**: 패키지 선택 → 자동 제안서 생성

**주요 기능**:
- [ ] 10개 대분류별 패키지 표시 (스타터/스탠다드/프리미엄)
- [ ] 패키지 상세 설명 (포함 서비스, 가격, 기대 성과)
- [ ] 패키지 조합 선택 (다중 선택 가능)
- [ ] 가격 자동 계산
- [ ] 커스터마이징 옵션 (추가 서비스, 할인)

**화면 요소**:
```
┌──────────────────────────────────────────────┐
│ 패키지 조합기                                  │
├──────────────────────────────────────────────┤
│ 퍼포먼스 마케팅                                │
│  [○ 스타터 50만]  [● 스탠다드 150만]  [ 프리미엄 350만] │
│ 설명: 메타, 구글 통합 운영, 월 ROAS 2.5~3.5배 목표 │
│                                              │
│ SNS 마케팅                                    │
│  [○ 스타터 40만]  [○ 스탠다드 120만]  [ 프리미엄 280만] │
│ ...                                          │
├──────────────────────────────────────────────┤
│ 총 월 투자: 150 + 120 + 60 = 330만원           │
│ [제안서 생성] [PDF 다운로드]                     │
└──────────────────────────────────────────────┘
```

**기술 요구사항**:
- 데이터베이스: 패키지 마스터 (10개 대분류 × 3단계)
- 계산 엔진: 가격 + 옵션 조합 로직
- 문서 생성: 제안서 템플릿 (HTML → PDF)

#### 3. 제안서 자동 생성
**목적**: 패키지 선택 → 1분 내 제안서 완성

**주요 기능**:
- [ ] 클라이언트 정보 입력 (회사명, 담당자, 예산)
- [ ] 패키지 선택 (위 카탈로그에서 선택)
- [ ] 자동 제안서 생성 (구글 문서 템플릿 활용)
- [ ] PDF 다운로드
- [ ] 이메일 자동 발송 (템플릿)

**필드**:
- 클라이언트명, 담당자, 연락처, 이메일
- 예산 규모, 시작일
- 추천 패키지 (자동 매핑)
- 기대 성과 (패키지별 자동 삽입)

**기술 요구사항**:
- 문서 생성: Google Docs API 또는 Docx 라이브러리
- 변수 치환: Mustache/Jinja2
- PDF 변환: wkhtmltopdf 또는 Playwright

#### 4. 계약 관리
**목적**: 제안서 → 계약서 → SOW 자동 생성 및 추적

**주요 기능**:
- [ ] 제안서 → 계약서 변환 (자동)
- [ ] SOW (Statement of Work) 자동 생성
- [ ] 서명 관리 (DocuSign 또는 유사 서비스)
- [ ] 계약 기간 추적 (시작/종료/갱신 알림)
- [ ] 문서 보관 (클라이언트별)

**화면 요소**:
```
┌──────────────────────────────────────────┐
│ 계약 관리                                  │
├──────────────────────────────────────────┤
│ 상태: [● 진행 중] [ 제안서] [계약서] [ SOW]    │
├──────────────────────────────────────────┤
│ 계약 기간: 2026-04-15 ~ 2026-10-15        │
│ 갱신 기한: 2026-07-15 (60일 남음)           │
│ [갱신 알림] [문서 다운로드] [PDF 인쇄]        │
└──────────────────────────────────────────┘
```

**기술 요구사항**:
- eSign API (DocuSign, Adobe Sign)
- 클라우드 스토리지 (Google Drive 또는 AWS S3)
- 자동화: Zapier 또는 커스텀 API

#### 6. 성과 리포트 자동 생성
**목적**: 월별 성과 리포트 자동 생성 → PDF 변환 → 이메일 발송

**주요 기능**:
- [ ] 월별 자동 리포트 생성 (월말 +1일)
- [ ] 리포트 구성 자동 완성
  - Executive Summary
  - KPI 대시보드
  - 채널별 분석
  - 크리에이티브 성과
  - 개선안 및 다음 달 계획
- [ ] PDF/PPT 자동 생성
- [ ] 이메일 자동 발송 (클라이언트에게)
- [ ] 클라이언트 포탈에 자동 업로드

**리포트 예시**:
```
┌─────────────────────────────────┐
│ 월간 마케팅 성과 리포트           │

├─────────────────────────────────┤
│ 1. Executive Summary             │
│  - 지출: 150만원                │
│  - 전환: 250건                  │

│                                 │
│ 2. 채널별 성과                   │
│  - 메타: 75만원 (ROAS 3.5배)     │
│  - 구글: 45만원 (ROAS 3.0배)     │
│  - 네이버: 30만원 (ROAS 2.5배)   │
│                                 │
│ 3. 크리에이티브 Top 3             │
│  - 이미지A: CTR 8.5%, CPA 5,500원│
│  - 영상B: CTR 6.2%, CPA 6,200원  │
│  - 이미지C: CTR 5.8%, CPA 6,800원│
│                                 │
│ 4. 개선안                       │
│  - 이미지A 예산 +20% 증액 제안   │
│  - 크리에이티브 신규 추가 (5월)   │
│  - 오디언스 확대 테스트 예정     │

```

**기술 요구사항**:
- 자동화: AWS Lambda + Scheduled Rule (월말 23:00)
- 문서 생성: Python Docx / Jinja2 (Word/PPT)
- PDF 변환: LibreOffice / Playwright
- 이메일: SendGrid / AWS SES (템플릿 사용)
- 스토리지: 클라우드 (Google Drive 자동 저장)

#### 7. 클라이언트 포탈
**목적**: 클라이언트가 실시간으로 성과를 조회

**주요 기능**:
- [ ] 로그인 (이메일 + 비밀번호)
- [ ] 실시간 성과 대시보드 (일일 업데이트)
- [ ] 월별 성과 리포트 다운로드
- [ ] 계약서, SOW, 제안서 다운로드
- [ ] 티켓 시스템 (문의 접수 → 답변)
- [ ] 공지사항 (새 기능, 유지보수 안내)

**화면 요소**:
```

│ 클라이언트 포탈 — 홍길동 쇼핑몰    │

│ └─────┘ └─────┘ └─────┘ └─────┘│
│                                 │
│ 최신 리포트                       │
```

**기술 요구사항**:
- 프론트엔드: React / Vue.js (반응형)
- 인증: JWT (토큰 기반)
- API: RESTful (클라이언트 권한 필터링)
- 데이터베이스: 클라이언트별 데이터 격리 (Row-level Security)

#### 8. 실행사 (벤더) 배정 시스템
**목적**: 서비스 유형 → 최적 벤더 자동 추천

**주요 기능**:
- [ ] 벤더 마스터 데이터 (38개 업체, 전문분야, 가격대)
- [ ] 서비스 유형별 벤더 매칭 (예: 메타 광고 → 에코마케팅, 이엠넷)
- [ ] 벤더별 성과 추적 (ROAS, CPA, 만족도)
- [ ] 벤더 선택 UI (자동 추천 + 수동 선택)
- [ ] 계약 자동 생성 (B2B 벤더와의 계약)

**매칭 로직** (예시):
```
퍼포먼스 마케팅 패키지 선택
  ↓
[가능한 벤더들]
- 에코마케팅: D2C 커머스 특화, 월 200~300만원대
- 이엠넷: 종합 퍼포먼스 (상장사), 월 300~500만원대
- 아이엔마케팅: 구글 공식 파트너, 월 150~250만원대
  ↓
[PM이 클라이언트 특성 고려해 선택]
→ 자동 벤더 계약서 생성
→ 벤더에게 요청사항 전달
```

**기술 요구사항**:
- 데이터베이스: 벤더 마스터 (capacity, 가격, 평점)

## 4. 기술 스택 제안

### 4-1. 백엔드
- **언어**: Python (Flask/FastAPI) 또는 Node.js (Express)
- **데이터베이스**: PostgreSQL (관계형) + Redis (캐시)
- **인증**: JWT + OAuth 2.0
- **API**: RESTful + GraphQL (선택)

### 4-2. 프론트엔드
- **프레임워크**: React.js / Vue.js
- **상태 관리**: Redux / Vuex
- **UI 라이브러리**: Material-UI / Ant Design
- **실시간 업데이트**: WebSocket (Socket.io)

- **모니터링**: DataDog / New Relic
- **백업**: AWS S3 + Glacier

## 7. 위험 및 완화 전략

### Risk 1: API 변경
**위험**: Meta/Google/Naver 등 API 구조 변경 → 수집 실패

**완화**:
- 각 플랫폼별 API 모니터링 (스크립트)
- 여러 데이터 소스 연동 (직접 API + 제3 분석 서비스 병용)
- API 버전 관리 (v1.0, v2.0 동시 지원)

---

### Risk 2: 데이터 보안
**위험**: 클라이언트 광고 계정 토큰 유출 → 신뢰 상실

**완화**:
- 토큰 암호화 (AES-256)
- 최소 권한 원칙 (읽기 전용 토큰)
- 정기 감사 (월 1회 보안 점검)
- 2FA 의무화

---

### Risk 3: 스케일링
**위험**: 100+ 클라이언트 동시 접속 → 시스템 다운

**완화**:
- 마이크로서비스 아키텍처 (각 기능별 독립 배포)
- 캐싱 전략 (Redis, CDN)
- 자동 스케일링 (Kubernetes)
- 부하 테스트 (월 1회)
